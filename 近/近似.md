'''近似'''或是'''逼近'''是指一個事物和另一事物類似，但不是完全相同。近似可以用在許多性質上（量、數值、影像或說明），是指幾乎一様，但沒有完全一様的情形。

近似最常用在[[數字|數字]]上，也常用在[[數學函數|數學函數]]、[[形狀|形狀]]及[[物理定律|物理定律]]中。

在科學上，會將一物理現象轉換為一個有相似結構的模型<ref>{{cite journal 
 | author=吳明珠 
 | title= 科學模型本質剖析：認識論面向初探
 | journal=科學教育月刊
 | volume=207 
 | pages=p2-8 
 | url=http://www.sec.ntnu.edu.tw/Monthly/97%28306-315%29/307-PDF/01-97039.pdf 
 | accessdate=2013-07-24|date=2008年4月}}</ref>，當準確的模型難以應用時，會用一個較簡單的模型來近似，簡化中間的計算，例如用[[球棒模型|球棒模型]]來近似實際化學分子中原子的分佈。當由於[[資訊|資訊]]不完整，無法確切陳述特定事物時，也可以用近似的方式處理。

近似的種類會依照可以取得的資訊、需要的準確程度及使用近似可以節省的時間及精力而定。

== 數學 ==
[[逼近理論|逼近理論]]（Approximation theory）是數學中的一個分支，是一種量化的[[泛函分析|泛函分析]]。[[丟番圖逼近|丟番圖逼近]]是用[[有理數|有理數]]來逼近[[實數|實數]]。當一個數的真正數值未知或難以獲得時，就可以用近似（即逼近）的方式處理。有時存在一些已知的近似值可以表示其真正數值，而又不會有太大的誤差，例如[[圓周率|圓周率]]π常簡寫為3.14159，或是√2用1.414來表示。

當使用數字的[[有效數字|有效數字]]很小時，也會出現數值逼近的情形，運算常會帶來[[捨入誤差|捨入誤差]]，因此會產生逼近。像[[對數表|對數表]]、[[計算尺|計算尺]]及計算器在計算大部份的運算時也都會有數值逼近。像電腦計算的結果就是以有限位數的有效數字來呈現，因此也有數值逼近，不過可以藉由設計．使其逼近誤差更低，產生更準確的結果<ref>[http://docs.oracle.com/cd/E19957-01/806-3568/ncg_goldberg.html Numerical Computation Guide]</ref>。在電腦處理時，當一個小數無法用有限位數的二進數小數表示時，就會產生數值逼近。

和函數逼近有關的是函數的[[漸近分析|漸近]]值，也就是當函數的一個或數個變數無限制的變大時，函數所對應的數值。例如級數(''k''/2)+(''k''/4)+(''k''/8)+...(''k''/2^''n'')會漸近等於''k''。可惜上述的關係沒有類似[[等號|等號]]的固定符號來表示。有些數學書籍是用≈表示逼近等於，用~表示表示漸近等於，但也有其他書籍的表示方式恰好相反。

另一個例子是在[[進化演算法|進化演算法]]中，為了加速收斂的速率所導入的{{link-en|适应度逼近|fitness approximation}}，可以針對{{link-en|适应度函数|fitness function}}建模，以選擇較佳的搜尋方式。

== 科學 ==
在科學實驗中也有逼近的情形．科學理論的預測可能會和實際量測的結果不同，其原因可能因為有一些實際情形下的因素，在理論中沒有考慮到。例如在考慮[[自由落體|自由落體]]的運動時．未考慮[[阻力|阻力]]對物體的影響，因此理論也是對實際情形的一種逼近。若因為量測技術的限制，使得量測值和實際值不同，此情形的量測值也是實際值的逼近。

在[[科学史|科学史]]上，許多定理會隨著時間演進，考慮更多的因素和影響，早期的定理也就成為後來定理的一個逼近。例如依照[[对应原理|对应原理]]，較新的定理會取代較早期的定理，在適當的條件下，二個的結果相同，但較新的定理可以考慮較多的因素或是適用在一些特別的情形，此時較早期的定理就是較新的定理的一個近似<ref>[http://www.britannica.com/EBchecked/topic/138678/correspondence-principle Encyclopedia Brittanica]</ref>，例如系統「大」的情況下，較早期的[[古典物理學|古典物理學]]可以認為是較晚期[[量子物理學|量子物理學]]的一個近似。

有些物體問題難以直接分析，或是在現有可有的解析工具下進展有限，因此利用近似可以在簡化問題的複雜程度下，得到夠精確的結果。例如物理學家多半會假設[[地球|地球]]為一[[球體|球體]]，既使有更精確的方式可以描述[[地球的形狀|地球的形狀]]，但假設[[地圓說|地球為一球體]]時，一些物理特性（像[[重力|重力]]）的計算會容易很多。

在分析多個星體圍繞一恆星運轉時（即[[多體問題|多體問題]]），也會用近似的方式處理。由於各星體之間都會有萬有引力，在計算上相當困難<ref>[http://plus.maths.org/content/mathematical-mysteries-three-body-problem The three body problem]</ref>。近似的解法是利用[[迭代|迭代]]方式進行，一開始先只假設恆星不動，不考慮恆星以外各星體之間的作用力，若需要更精確的結果，則以第一次計算的位置為準，在考慮更多作用力的情形下再進行一次迭代，一直到有夠精確的結果出現為止。利用[[微擾理論|微擾理論]]來修正誤差，可以得到更精確的結果。

在[[最佳化|最佳化]]演算法中，有些問題的最佳解很不容易取得，或是[[時間複雜度|時間複雜度]]太高，此時可以用[[近似演算法|近似演算法]]，設法找出夠好的解，而不一定是最佳解。

== 符號 ==
一般會用波浪狀或有加點的等點來表示<ref>{{cite web| title =Mathematical Operators – Unicode| url =http://www.unicode.org/charts/PDF/U2200.pdf| accessdate =2013-04-20}}</ref>：
* <span style="font-size: 150%;line-height:50%;" lang="en">≈</span> (Unicode 2248), 表達近似的是「[[波浪|波浪]]」等號。
* <span style="font-size: 150%;line-height:50%" lang="en">[[≃|≃]]</span> (Unicode 2243), <span lang="en">≈</span> 和 <span lang="en">=</span> 的混合，也用於代表[[漸近線|漸近於]]
* <span style="font-size: 150%;line-height:50%" lang="en">[[≅|≅]]</span> (Unicode 2245), 另一個 <span lang="en">≈</span> 和 <span lang="en">=</span> 的混合，有時用來代表[[同構|同構]]、[[同余关系|同余关系]]又或幾何學的[[全等|全等]]。
* <span style="font-size: 150%;line-height:50%" lang="en">[[~|~]]</span> (Unicode 007E), 有時用來代表[[比例|正比]]、和[[等價關係|等價關係]]有關、幾何學的[[相似|相似]]，又或代表[[隨機變數|隨機變數]]根據[[概率分佈|概率分佈]]的分佈情況。
* <span style="font-size: 150%;line-height:50%" lang="en">≒</span> (Unicode 2252), 用於[[日文|日文]]和[[韓文|韓文]]
