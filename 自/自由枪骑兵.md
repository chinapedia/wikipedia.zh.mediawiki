{{Vgname|自由枪骑兵|en=Freelancer|cn=自由枪骑兵|tw=星際遊俠}}是一部[[太空模拟游戏|太空战斗经营模拟]][[电脑游戏|游戏]]，[[Digital_Anvil|Digital Anvil]]制作，[[Microsoft_Game_Studios|Microsoft Game Studios]]发行。本于1999年由[[Chris_Roberts_(film_producer)|克里斯·罗伯茨]]（Chris Roberts，Digital Anvil创始人）首次公开，但由于制作计划数次延期以及制作组被[[微软|微软]]收购，游戏直到2003年才正式发售。

本游戏允许玩家驾驶一架[[星际战机|星际战机]]在数十颗[[恒星|恒星]]及其[[行星系|行星系]]之间游历。玩家可以作为[[商船队|商船队]]护卫与敌对的战机进行格斗，也可以投身于[[海盗|海盗]]事业。在停靠于空间站与行星时，玩家可以寻找雇主或进行贸易。在[[单人游戏|单人游戏]]中玩家扮演[[自由职业者|自由职业者]]（本作由此得名）'''Edison Trent'''，从[[外星人|外星人]]手中拯救整个[[天狼星|天狼星]]域。本作的剧情是线性的，但随着剧情的发展对玩家行动的限制将一一解除，直到单人游戏[[通关|通关]]后再无任何限制。在[[多人游戏|多人游戏]]下，玩家从一开始就是完全自由的。

罗伯茨本来承诺提供更多的游戏体验，例如支持数千玩家在线的多人模式，自动飞行控制和动态的经济体系。但最终发售的版本并未具备这些功能。本作最初公布的技术展示非常吸引人，但由于制作组被微软收购以及罗伯茨的离职，导致一些评论认为本作可能沦为[[雾件|雾件]]。游戏最终发布后，评论认为本作虽然品质不俗，但与当初的承诺仍有差距。

===人物===
[[Image:Freelancer_Characters.jpg|thumb]]
玩家在单人游戏中扮演的人物是'''Edison Trent'''。制作组为了尽量详细地塑造这个人物而放弃了让玩家自订角色外形的系统。<ref>{{cite web
  | author = Jorg Neumann
  | coauthors = Adam Foshko
  | title = Freelancer Strategy Guide (PC)
  | publisher = [[IGN|IGN]]
  | date = n.d.
  | url = http://guides.ign.com/guides/11219/
  | accessdate = 2007-11-13 }}
</ref>但玩家在行为方式上却有相当的自由，例如可以成为商人，海盗，或[[赏金猎人|赏金猎人]]。 <ref name="ignint2k" />随着剧情发展，玩家得以结识几位[[NPC|NPC]]人物，例如Liberty保安部队（Liberty Security Force，LSF）指挥官Jun'ko Zane，[[外星文明考古学家|外星文明考古学家]]Roland Quintaine教授，以及某[[秘密结社|秘密结社]]的领导人Casper Orillion。但与这些NPC的交流则只被限制为接受或暂时拒绝他们的委托。主角与其他人物的互动大多通过[[即时演算|即时演算]][[过场动画|过场动画]]表现。 

游戏中的NPC各自属于不同的组织派别，例如警察、海盗、公司企业，或政府机关。<ref name="ignhands" />每个组织都有各自的业务和使命，它们互相之间也有好恶关系。如果玩家摧毁某一组织的飞船，就会招致该组织及其盟友的怨恨。同样，玩家也可以籍由摧毁某组织的敌人来换取该组织及其盟友不同程度的好感。如果与一个特定组织关系恶劣，玩家将遭受其成员的拦截，并会被拒绝在该组织经营的空间站停泊。<ref name="freely" />玩家可以通过行贿来快速改善与特定组织的关系。<ref name="ignhands" />一批专业演员，例如 Ian Ziering (《[[飞越比佛利|飞越比佛利]]》), [[Jennifer_Hale|Jennifer Hale]] (《[[银河战士|银河战士]]》游戏系列), [[乔治·竹井|乔治·竹井]]（《[[星际旅行|星际旅行]]》），以及[[John_Rhys-Davies|John Rhys-Davies]] (《[[夺宝奇兵|夺宝奇兵]]》 电影和 《[[旅行者：平行世界|旅行者：平行世界]]》)为本游戏献声。次要的NPC和背景人声主要由业余演员配音，多人游戏因为没有主线剧情，玩家只能遇见这些次要角色。<ref name="spyrev" />

==开发历程==
[[Image:Freelancer_Bretonia_City_Concept.jpg|right]]
早在1997年，自[[Chris_Roberts_(film_producer)|克里斯·罗伯茨]]构思《[[银河飞将|银河飞将]]》之时就开始着手策划本游戏。<ref name="robertsgame" />他试图创造一个游戏历程与过场动画无缝连接、单人游戏与多人游戏共用的复杂世界，还带有简单易用的新型[[用户界面|用户界面]]。1999年，本作在GameStock（[[微软|微软]]主办的游戏业者年会）正式以《自由枪骑兵》之名公布。<ref>{{cite news
  | last = Werner
  | first = Nash
  | title = Gamestock '99: Microsoft's love-in
  | publisher = [[CNN.com|CNN.com]]
  | date = 1999-03-12
  | url = http://www.cnn.com/TECH/computing/9903/12/gamestock.idg/index.html
  | accessdate = 2007-11-22 }}
</ref>罗伯茨声明本作将于2000年秋季发售，并公开了一段本作的展示视频。<ref name="robertsgame" />游戏中的世界将充满对[[宏观经济|宏观经济]]造成影响的市场活动，城市中各类运输工具川流不息且具备气候变化，电脑控制的商户即使是在玩家视线之外也会自行进行进出口贸易，而数千玩家与这个持续存在的世界的互动，也会成为影响宏观经济的因素之一。玩家可以购买并建设他们自己的空间站。游戏的控制界面非常友好，只需鼠标便足以操控一切。游戏引入了神经网络（Neuronet，类似[[头盔显示器|头盔显示器]]）概念，玩家可以方便地启动[[AI|AI]][[自动导航|自动导航]]系统——只需按一个键玩家的星际战机就能自动尾随敌机，让玩家集中精力进行射击；另一个按键则让战机进入自动回避机动，逃离威胁。游戏将鼓励玩家赚钱升级他们的神经网络。罗伯茨还声称多人游戏部分的内容将单独发售。同年，《自由枪骑兵》参加了[[E3|E3]]展会，并获得了数个奖项。但是，当罗伯茨声明游戏的发售将推迟到2001年末时，本作开始了它的[[跳票|跳票]]历程。 <ref name="shoe" />即便如此，演示视频中展现出的惊人创意仍旧使[[Computer_Gaming_World|Computer Gaming World]]将本作评价为“将引发游戏产业的革命”。 <ref name="change">{{cite journal

《自由枪骑兵》最初公布的演示大获好评，在1999年度的E3获得四项[[游戏评论家奖|游戏评论家奖]]，包括当年度的最佳游戏奖。早期的演示视频和测试版也受到好评，只有一些对罗伯茨那些过于激进的理念的怀疑。<ref name="first" /><ref name="online" />虽说此后不断的跳票和制作公司遭收购使本作品质受到质疑，<ref name="anvilru" />最终发售的版本获得的多是正面评价。 

本作的图像效果被认为属于中流，但运行时的帧数却很稳定。<ref name="atomic" />FiringSquad评价道，虽然图像技术稍显落后，但艺术设计填补了这一瑕疵。 GameCritics.com和Skoar.com很喜爱本作宇宙的设计。<ref name="skoar" />AtomicGamer和[[GameSpot|GameSpot]]则报怨机械设定缺乏趣味，主力舰虽然大过星际战机，但对于一部2003年游戏来说仍嫌小气。<ref name="atomic" />Game Nation TV评价道，本作中所有势力拥有相似的风格，让“《[[星际旅行|星际旅行]]》系列看起来像异形动物园”。<ref name="gntv" />[[GameSpy|GameSpy]]认为，游戏中的每个行星系各有不同的视觉外观，但从功能上来说则是雷同的，这会降低玩家继续探索的兴趣。<ref name="spyrev" />分属不同民族的人物本该具备的文化差别也被清一色的美国口音所淡化。 <ref name="critic" />

Skoar.com称赞本作创造了一个非常适合[[角色扮演|角色扮演]]的非线性游戏体验，<ref name="skoar" />但其他评论则认为本作的世界不如早期公开的富于变化。即使互相之间打得你死我活，国家边界和海盗的势力范围从不改变。<ref name="gsrev" />NPC的活动也被削减为简单地在贸易航路上往来，按照固定的路线巡逻。游戏的经济体系是固定的，从头到尾物价几乎不会变动，只是有时随着新闻事件波动一下。 <ref name="spyrev" /><ref name="euro" /><ref name="cgw">{{cite journal
 |last=Nguyen 
 |first=Thierry 
 |title=Freelancer

</ref>玩家永远只能扮演一个独来独往的飞行员，不断地维修飞船、购买消耗品、进行小额贸易。无法雇佣[[僚机|僚机]]或护卫，也不能建立自己的船队与基地。玩家的人物没有个性的性格特征，对话总是固定地让玩家选择接受或回绝任务，<ref name="ignrev" />不过IGN认为游戏中无处不在的细节背景资料让这世界不那么无趣。尚有评论批评分支任务缺乏变化，全部要求玩家在一个指定地点击落敌机，互相之间只是稍有区别。 <ref name="stratos" /><ref name="spyrev" /><ref name="critic" /><ref name="ugo">{{cite web

Skoar.com和[[Stratos_Group|Stratos Group]]同意丰富的过场动画成功构建了有趣的单人游戏剧情，<ref name="stratos" /><ref name="skoar" />不过这些过场完全无法取消，即使是对人物神态和表情有传神表现，在多次观看后也变得让人无法忍受。AtomicGamer则发现有时人物动作与声音脱节，造成人物对不上口型。<ref name="atomic" />由专业演员如Ian Ziering担纲的配音被评价为非常成功，使人物生机勃勃。<ref name="ignrev" /><ref name="spyrev" /><ref name="allgame" />但只有一位配音演员为几乎所有随机遭遇中的男性角色配音，他单调平淡的表现为GameSpy，GameSpot，和[[Eurogamer|Eurogamer]]所诟病。游戏中动态对话系统为NPC角色创造的结结巴巴的对答也无助于改善游戏体验。<ref name="stratos" /><ref name="skoar" /><ref name="euro" />虽然游戏在行星与空间站的设计、人物的动态表情和NPC的随机对答中下了功夫，[[Tom_Chick|Tom Chick]]认为它们仍旧让他失去了进行交易或接受任务的兴趣，<ref name="fsrev" />而上述一切在完成剧情的过程中是无处不在的。其他一些评论不满主线剧情时常强行中断他们当前正从事的事务，命令他们前往指定地点去触发主线任务。<ref name="euro" /><ref name="atomic" />Skoar.com却认为正是这一点让玩家不至于过早陷入千篇一律的自由任务中。<ref name="skoar" />对剧情持正面观点的评价认为结局来得太过突然，让他们缺乏进一步的游戏目标，并发现结局后的游戏内容缺乏变化。<ref name="cvgrev" />

《自由枪骑兵》键盘鼠标配合的操控方式获得了好评。相比一般的[[太空模拟游戏|太空模拟游戏]]，本作的操纵方式更接近于[[第一人称射击|第一人称射击]]。结合方便易懂的鼠标点击系统，作战杀敌也显得简单了一些。<ref name="stratos" /><ref name="allgame">{{cite web
  | author = Michael L. House
  | title = Freelancer
  | publisher = [[Allgame|Allgame]]
  | date = n.d.
  | url = http://www.allgame.com/cg/agg.dll?p=agg&sql=1:24090~T1
  | accessdate = 2007-10-26 }}
</ref>GameSpy评价道，游戏中的战斗非常有趣，有助于延长多人游戏的寿命。IGN指出这套控制系统虽然符合游戏探索发现与刺激战斗的主题，但会让战斗手段显得单一。 游戏中很难作出剧烈的转向与回旋机动，让本作中的[[dog_fight|近身缠斗]]成为单调的追或被追，与同类型的其他游戏相比显得苍白无力。<ref name="gntv" /><ref name="hist" />游戏中不允许雇佣僚机，对战机的改装受到限制，所有战机的速度完全一样，[[火控系统|火控系统]]也过于简单，被GameSpot批评为战术上过于肤浅。Skoar.com称赞游戏中的AI会果断打散队形试图对玩家实施迂回，<ref name="skoar" />[[Computer_and_Video_Games|PC Zone]]则批评AI在混战中只懂得围攻玩家。[[UGO_Networks|UGO]]也发现，由于友军AI不够积极，击落最后一架敌机结束战斗的时常是玩家本人。<ref name="ugo" />GameCritics.com很满意本作的自动寻路系统，认为游戏中简单易用的未来版导航系统非常真实可信。AtomicGamer则指出过大的目标箭头（用于指示离开屏幕范围的目标）配上难看的字体，对战斗并无帮助。 

FiringSquad认为，《自由枪骑兵》的多人模式是“本作最大的成功与失败”。该模式的优点在于玩家可以完全自由活动，或与他人合伙赚钱，或贩卖船只和装备。但是，游戏中完全固定的经济体系和国际形势，加上可选的商品非常有限， Game Nation TV直言多人模式只不过是没有主线剧情的单机模式而已。<ref name="gntv" />由于玩家无法自创人物，人人都是"Edison Trent"，导致玩家不能进行个人交往。不过，即使有着上述限制，Stratos Group仍认为玩家在多人游戏中能享受充分的乐趣。因为真人玩家到底不是AI，他们可以密切配合完成独自一人不能成功的任务。<ref name="stratos" />但是，由于单人游戏与多人游戏的自由任务是完全一样的， IGN和GameCritics质疑即使有朋友相伴，将这些相同的内容重玩多次有否意义。IGN更指出既然玩家不能将成就带到其他服务器，使努力赚取大量财富、购买高端装备的行动显得毫无价值。AtomicGamer则赞扬了多人游戏不易[[跳帧|跳帧]]的稳定性。<ref name="atomic">{{cite web

总而言之，《自由枪骑兵》被大多数评论者认为是一部技术成熟，可玩性高、颇有乐趣的游戏。<ref name="gsrev" /><ref name="ugo" />游戏使用了“菜鸟主角经历艰辛最后拯救世界”的剧情套路，允许玩家在结局后继续游戏。本作作为[[太空模拟游戏|太空模拟游戏]]表现中规中矩，获得了不错的销量，被评价为物有所值。<ref name="ignrev" /><ref name="armchair" />但是本作削减或放弃了部分早已承诺的游戏特性，未能满足支持者的全部期待。<ref name="spyrev" /><ref name="hist">{{cite web
