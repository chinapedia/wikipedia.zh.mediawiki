==磁盘文件系统==

磁盘文件系统通常面向块。文件在面向块的文件系统中往往是连续的块，一般主要是完全随机的读取、写入和修改操作。

=== 为闪存、固态硬盘优化的文件系统 ===
{{Main|闪存文件系统}}

固态介质（如[[闪存|闪存]]）在接口上类似磁盘，但具有不同的问题。在底层上，它们需要如[[耗損平均技術|耗損平均技術]]和不同的[[错误检测和纠正|错误检测和纠正]]算法的特殊处理。通常设备（如[[固态硬盘|固态硬盘]]）会在其内部处理此操作，因此一个可以使用常规的文件系统。但是，某些专用的设备（嵌入式设备、工业应用等）需要使用一个为纯闪存优化的文件系统。

* {{tsl|en|Cache Accelerated Sequential Layout|CASL}} 是一个{{tsl|en|Nimble Storage}}设计的文件系统，它使用固态设备来缓存传统硬盘设备。

* [[ExFAT|ExFAT]] - [[微软|微软]]专有系统，面向闪存卡（另见{{tsl|en|XCFiles}}，一个用于[[溫瑞爾系統|Wind River]] [[VxWorks|VxWorks]]和其他嵌入式操作系统的exFAT实现）
* [[快閃記憶體檔案系統|快閃記憶體檔案系統]] - 用于SSD的内部文件系统。
* [[F2FS|F2FS]] – Flash-Friendly File System。一个开源Linux文件系统，[[三星集团|三星集团]]于2012年引入。<ref>{{cite web |url=http://www.phoronix.com/scan.php?page=news_item&px=MTE5OTY |title=Samsung Introduces New Linux File-System: F2FS |publisher=phoronix.com |author=Michael Larabel |date=2011-10-05 |accessdate=2012-12-07}}</ref>
* [[闪存|FFS2]]（想必之前还有FFS1），最早的闪存文件系统之一。[[微软|微软]]于1990年代早期开发和取得专利。<ref>{{cite web|url=http://patft.uspto.gov/netacgi/nph-Parser?Sect1=PTO2&Sect2=HITOFF&p=1&u=%2Fnetahtml%2FPTO%2Fsearch-bool.html&r=1&f=G&l=50&co1=AND&d=PTXT&s1=5392427.PN.&OS=PN/5392427&RS=PN/5392427 |title=United States Patent: 5392427 |publisher=Patft.uspto.gov |date= |accessdate=2012-06-15}}</ref>

=== 面向记录文件系统 ===

在{{tsl|en|Record-oriented filesystem|面向记录文件系统}}中的文件是存储为一个[[记录|记录]]的集合。它们通常与[[大型计算机|大型计算机]]和[[小型计算机|小型计算机]]的操作系统相关。程序读取和写入整个记录，而不是字节或任意字节范围，并且可以查找一个记录边界而不是记录内。更复杂的面向记录文件系统与简单的[[数据库|数据库]]而非其他文件系统有更多共同点。
*{{tsl|en|Files-11}} – 早期版本以“记录”为导向；后来增加了对“流”的支持

=== 共享磁盘文件系统 ===
共享磁盘文件系统，也称共享存储文件系统、[[SAN文件系统|SAN文件系统]]、[[集群文件系统|集群文件系统]]、群集文件系统。主要用于所有节点直接访问文件系统所在[[块_(数据存储)|块存储]]设备的[[存储区域网络|存储区域网络]]。这使得节点可以失败而不影响从其他节点的文件系统访问。共享磁盘文件系统通常使用在一个{{tsl|en|high-availability cluster|高可用性集群}}，与硬件[[RAID|RAID]]结合使用。共享磁盘文件系统通常不超过64或128节点的规模。

* {{tsl|en|Microsoft Office Groove}}共享工作区，用于DoHyki

=== 分布式容错文件系统 ===

分布式节点数据[[容错|容错]]复制（在服务器或服务器与客户端之间），面向[[高可用性|高可用性]]和[[在线和离线|离线]]（断开）操作。

* [[Coda|Coda]]，出自[[卡内基梅隆大学|卡内基梅隆大学]]，着重于带宽自适应操作（包括已断开操作），使用一个客户端侧缓存，用于移动计算。它是一个AFS-2的继任者。它可以[[GNU通用公共许可证|GPL]]用于[[Linux|Linux]]。

* {{tsl|en|Tahoe-LAFS}}是一个开源、安全、分布式、容错的文件系统，采用加密为基础，作为一个最小权限复制的设计。

=== 分布式并行文件系统 ===

分布式[[并行计算|并行]]文件系统是将数据条带化传往服务器以提供高性能。它们通常用于[[超级计算机|超级计算机]]。

部分分布式并行文件系统使用{{tsl|en|object storage device|对象存储设备}}（OSD）（也称OST）将数据块的[[元数据|元数据]]整合到中央服务器。

* {{tsl|en|Parallel Virtual File System}} (PVFS, PVFS2, {{tsl|en|OrangeFS}})。为存储虚拟系统镜像开发，着重于非共享写入优化。可以[[GNU通用公共许可证|GPL]]用于[[Linux|Linux]]。

=== 对等文件系统 ===

其中一些可能被称为{{tsl|en|cooperative storage cloud|合作存储云}}。

* {{tsl|en|Cooperative File System}} 是一个基于{{tsl|en|Chord (distributed hash table)|Chord}} DHT的只读文件系统。<ref>{{cite web |title= a Cooperative File System |author= Frank Dabek |publisher= MIT |date= September 5, 2001 |url= http://ftp.lcdf.org/papers/chord:dabek_thesis/dabek.pdf |accessdate= May 30, 2013 |deadurl= yes |archiveurl= https://web.archive.org/web/20131004230200/http://ftp.lcdf.org/papers/chord%3Adabek_thesis/dabek.pdf |archivedate= 2013年10月4日 }}</ref>
* {{tsl|en|Cleversafe}} 使用Cauchy [[里德-所罗门码|里德-所罗门码]] {{tsl|en|information dispersal algorithms|信息分散算法}}分散内容到不可辨识的程度，然后通过安全的互联网连接分发到多个存储位置。
* {{tsl|en|Infinit (file system)|Infinit}} 是一个对等文件系统，可以预置部署，或者通过一个不同性质的存储资源集：本地磁盘、NAS以及甚至基于云的资源。
* [[Ivy_(文件系统)|Ivy]] 是一个多用户读写的对等文件系统。Ivy没有中心化或专用组件，它提供实用的完整性属性，而不需要用户完全信任底层对等存储系统或其他该文件系统的用户。<ref>http://pdos.csail.mit.edu/ivy/osdi02.pdf</ref>
* {{tsl|en|Pastis (file system)|Pastis}}文件系统 是一个法国的对等文件系统，使用Java开发
* {{tsl|en|NimbusFS}} 是一个基于DHT的云文件系统，内置复制和x509证书验证，发布在[[創作共用授權條款|創作共用授權條款]]授权。
* [[IPFS|IPFS]] InterPlanetary File System，是一个P2P、全球分布式内容可寻址文件系统。
