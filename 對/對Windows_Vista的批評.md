'''[[Windows_Vista|Windows Vista]]'''為[[微軟|微軟]]於[[2006年11月|2006年11月]]所公布的一套[[作業系統|作業系統]]，並於2007年1月30日正式發售。儘管微軟在[[美國|美國]][[紐約|紐約]]等地舉行了多場的宣傳活動為其造勢，微軟首席執行官[[史蒂夫·巴爾默|史蒂夫·巴爾默]]親自在活動上表示：「Windows Vista就是下一代家庭互聯式娛樂的核心，它的出現是下一代家庭互聯式娛樂開始。」而微軟董事長[[比爾·蓋茲|比爾·蓋茲]]亦宣稱新作業系統將比以前所有計算平台都要優秀，當消費者們看到Windows Vista時，會讓他們帶來驚喜不已，<ref name = "PConline">{{Cite web |url= http://news.pconline.com.cn/hy/vista/0702/980006.html |title= Vista上市市场反应冷淡 取代WinXP尚须时日 |author= vigarbuaa |date= 2007年2月1日 |publisher= [[太平洋電腦網|太平洋電腦網]] |language= zh-hans |accessdate= 2010年12月19日 |quote=  }}</ref> 但無論是在[[隱私|隱私]]、[[信息安全|安全]]、效能、[[翻譯|翻譯]]以及[[產品啟動|產品啟動]]等各方面，Vista反而受到許多使用者或評論者的批評。

=== 速度 ===
[[2007年1月|2007年1月]]，[[Tom's_硬體指南|Tom's 硬體指南]]發布其十大應用測試結果中表明：[[Windows_Vista|Windows Vista]]的[[應用程序|應用程序]]執行速度和一般的[[Windows_XP|Windows XP]]相比，在相同的硬件配置條件下更慢。<ref>{{Cite web |url= http://www.tomshardware.com/reviews/xp-vs-vista,1531.html |title= Windows XP vs. Vista: The Benchmark Rundown |author= Patrick Schmid |date= 2007年1月29日 |publisher= [[Tom's_硬體指南|Tom's 硬體指南]] |language= en |accessdate= 2010年12月18日 |quote=  }}</ref> Tom's 硬體指南認為Vista性能下降的原因是因為Vista在其巨幅的設計變更中，並沒有特別考慮實際運作的功效。但也有如同Standard Performance Evaluation Corporation等人指出結果「不應該是這樣如此比較」，理由為測試電腦的製造當時是在Windows XP推出，即使是要比較測試也不應該是以相同的硬體配置來比較；由於硬體規格的问题，也可能導致出現「無效的比較」資料。<ref>{{Cite web |url= http://www.spec.org/gwpg/publish/vista_paper.html |title= Understanding the impact of Windows Vista on SPECviewperf performance measurement |author= Ian Williams |date= 2007年5月29日 |publisher= Standard Performance Evaluation Corporation |language= en |accessdate= 2010年12月18日 |quote=  }}</ref>

如在[[WinRAR|WinRAR]]和[[Adobe_Photoshop|Adobe Photoshop]]的兩次實際應用測試上，數據上Windows Vista反而分別快了21.8％和5.5％左右，但實際上卻不然。<ref name = "Power">{{Cite web |url= http://www.tomshardware.com/reviews/xp-vs-vista,1531-11.html |title= Power Consumption Tests |author= Patrick Schmid |date= 2007年1月29日 |publisher= [[Tom's_硬體指南|Tom's 硬體指南]] |language= en |accessdate= 2010年12月18日 |quote=  }}</ref> 而其他的三項應用程式分別運行在XP和Vista中時，只有2％之間的速率差異，幾乎可以忽略不計。Tom's 硬體指南報告在這次測試中承認「Windows Vista嚴重缺乏[[OpenGL|OpenGL]]圖形庫的支援」，也因此報告中建議直到製造商寫出可有效運作的驅動程序時，才需要把Windows XP替換成Windows Vista。<ref name = "Power"/>

許多中低效能的電腦常因為預先裝設Windows Vista系統，速度異常緩慢；部分[[筆記型電腦|筆記型電腦]]製造商已經表示願意協助使用戶將筆記型電腦「降級」到Windows XP的版本。<ref>{{Cite web |url= http://www.engadget.com/2009/02/16/microsoft-gets-sued-over-windows-xp-downgrade-fees/ |title= Microsoft gets sued over Windows XP downgrade fees |author= Donald Melanson |date= 2009年2月16日 |publisher= [[癮科技|癮科技]] |language= en |accessdate= 2010年12月18日 |quote=  }}</ref>

=== 檔案操作性能 ===
[[2006年11月|2006年11月]]首次發布的Windows Vista中，在執行如複製、刪除等檔案操作時，速度皆比其他作業系統慢。當使用者有一龐大檔案需要從一台電腦遷移到另一台時，這項轉移工作反而變得十分困難。由於無法有效地執行基本的檔案操作，Windows Vista收到許多使用戶的強烈批評。<ref>{{Cite web |url= http://social.technet.microsoft.com/forums/en-US/itprovistadesktopui/thread/30c9c7f3-2cc4-426a-9db9-885086ff183e/ |title= calculating time remaining moving, deleteing, copying files verry slow |author=  |date= 2010年7月14日 |publisher=  [[Microsoft_TechNet|Microsoft TechNet]] |language= en |accessdate= 2010年12月18日 |quote=  }}</ref> 六個月後微軟證實Windows Vista的確存在這些問題，並藉由包含Service Pack 1等[[Windows_Update|Windows Update]]工作，以增進檔案運作效能和可靠性。<ref>{{Cite web |url= http://support.microsoft.com/?kbid=938979 |title= 現已提供更新，可以改善 Windows Vista 的效能和可靠性 |author=  |date= 2008年1月17日 |publisher= Microsoft技術支援 |language= zh-hant |accessdate= 2010年12月18日 |quote=  }}</ref><ref>{{Cite web |url= http://windowsteamblog.com/windows/archive/b/windowsvista/archive/2007/08/28/windows-vista-service-pack-1-beta-whitepaper.aspx |title= Windows Vista Service Pack 1 Beta White Paper |author= Brandon LeBlanc |date= 2007年8月28日 |publisher= Windows Vista Team Blog |language= en |accessdate= 2010年12月18日 |quote=  }}</ref> 但還是仍有報告指出，和其他各版本於同階段時所提高的性能相比，Windows Vista從原始版本加強到Service Pack 1版本的運作水平仍是沒有比Windows XP作業系統特別突出；<ref>{{Cite web |url= http://www.zdnet.com/blog/hardware/vista-sp1-vs-xp-sp2-benchmarked/1332 |title= Vista SP1 vs. XP SP2 - Benchmarked |author= Adrian Kingsley |date= 2008年2月15日 |publisher= [[ZDNet|ZDNet]] |language= en |accessdate= 2010年12月18日 |quote=  }}</ref> 但這次批評，也有人抱持著相反的意見。<ref>{{Cite web |url= http://www.zdnet.com/blog/bott/another-take-on-vista-vs-xp-benchmarks/369 |title= Another take on Vista vs. XP benchmarks |author= Ed Bott |date= 2008年2月19日 |publisher= [[ZDNet|ZDNet]] |language= en |accessdate= 2010年12月18日 |quote=  }}</ref>

== 安全性 ==
[[微軟|微軟]]原本打算強調Vista升級後的安全性能，如[[BitLocker驱动器加密|BitLocker]]加密技術，微軟並指出Vista的安全漏洞比上市第一年時的XP減少了一半以上。而微軟副[[總裁|總裁]][[布拉德·布魯克斯|布拉德·布魯克斯]]甚至認為即便是第一年發布的版本，Vista也是當時最安全的商用作業系統，而「[[蘋果公司|蘋果]]從未這麼表示」，不過微軟對此也沒有舉出相應的證據。

「對於Vista我們很自信，它是微軟到目前為止最為安全的作業系統，……」微軟的一位負責Vista安全的高級經理說：「不過，有一點大家必須認識到，那就是沒有100%安全的系統……」「Vista對於攻擊沒有免疫力，因此，我們如果對它抱有這樣的希望也是不公平的。」<ref name = "ZOL">{{Cite web |url= http://news.zol.com.cn/54/542269.html |title= 專業IT媒體文章：Windows Vista有多安全？ |author= 《[[計算機世界|計算機世界]]》 |date= 2007年4月16日 |publisher= ZOL |language= zh-hans |accessdate= 2010年12月19日 |quote=  }}</ref> 產業分析公司Burton Group的分析師[[Pete_Lindstrom|Pete Lindstrom]]則說：「微軟聲稱在安全性方面Vista領先XP很多，但市場將會檢驗它是否重要。值得強調的是，新系統所暴露的問題還很少，我們對此相關紀錄還很短。」<ref name = "PConline"/>

=== 使用者帳戶控制 ===
大部分使用者批評新的[[使用者帳戶控制|使用者帳戶控制]]（User Account Control，UAC）其安全技術，因為許多第三方程序設計時並沒有遵循[[最小許可權原則|最小許可權原則]]，而需要使用者以[[系統管理員|系統管理員]]身份來運行，導致在每次安裝應用程式或變更設定時，都會彈出UAC的提示對話框，帶來了不便及麻煩。之所以沒達到原本預設的目的是因為過去一段時間以來，[[微軟|微軟]]雖然一直建議廠商將程式編寫為可由普通用戶身份直接運行。但是在幾乎所有的使用戶默認情況下，儘管使用者只有在完成某項任務時必須行使管理員權限，比如安裝設備[[驅動程序|驅動程序]]或者修改註冊表時，但是過去[[Windows|Windows]]系統沿襲了[[DOS|DOS]]系統的一些特性，兩者本質上為單用戶系統，所以系統中的用戶默認都是管理員。甚至在微軟首個多用戶客戶端[[Windows_XP|Windows XP]][[作業系統|作業系統]]中，用戶登錄時仍然習慣性地默認是系統管理員，即便他只需要完成一些最普通的任務。<ref name = "ZOL"/> 這導致在以前版本的Windows中管理員的權限跟一般使用戶的權限界線是相當模糊的，許多開發商也因而錯誤地設定這些權限。<ref>{{Cite web |url= http://technet.microsoft.com/en-us/magazine/cc160944.aspx |title= Problems of Privilege: Find and Fix LUA Bugs |author= Aaron Margosis |date=  |publisher= [[微軟|微軟]] |language= en |accessdate= 2010年12月18日 |quote=  }}</ref>

儘管立意上，是要改變過去這種狀況。過去這方法雖然讓工作站非常容易管理，但對安全而言是一個漏洞。如果用戶以管理員的身份登錄時，一旦[[電腦蠕蟲|電腦蠕蟲]]、[[特洛伊木馬_(電腦)|特諾伊木馬]]等入侵成功後，就會擁有最高的權限，從而給用戶帶來很大的破壞。在加上微軟對用戶權限管理的疏忽，讓軟件開發商們可以很隨意地採用一些不安全的編程方法，而讓狀況變得更糟。也因此許多Windows上的應用只有在管理員權限下才能運行，這就意味著系統此時於一種最不安全的狀態下運行。<ref name = "ZOL"/> UAC試圖對這種不良的習慣進行修正，在UAC模式下大多數軟體默認運行在低用戶權限的狀態之下，但如果某個應用進行的某個操作需要管理員的權限，UAC會彈出一個對話框，要求用戶把應用提升到一個更高級的權限。<ref name = "ZOL"/><ref name = "people">{{Cite web |url= http://it.people.com.cn/GB/42892/42925/5598231.html |title= 批评与赞美 走进Vista系统的台前幕后 |author=  |date= 2007年4月11日 |publisher= [[人民網|人民網]] |language= zh-hans |accessdate= 2010年12月18日 |quote=  }}</ref><ref>{{Cite web |url= http://channel9.msdn.com/Shows/Going+Deep/UAC-What-How-Why |title= UAC - What. How. Why.（相關影片） |author= Charles |date= 2007年3月5日 |publisher= channel9 |language= en |accessdate= 2010年12月18日 |quote=  }}</ref><ref>{{Cite web |url= http://technet.microsoft.com/en-us/library/cc709618(WS.10).aspx |title= Notable Changes in Windows Vista Service Pack 1 |author=  |date=  |publisher= [[微軟|微軟]] |language= en |accessdate= 2010年12月19日 |quote=  }}</ref>

但Vista中的UAC並不完善，且存在一些漏洞可以被利用。就和Windows的其他作業系統一樣，Vista中軟體的安裝者總是具有管理員的權限。[[賽門鐵克|賽門鐵克]]的安全分析師[[Ollie_Whitehouse|Ollie Whitehouse]]指出，Vista中自身的一些應用程序也可能被利用拿來破壞UAC的保護作用。<ref name = "ZOL"/> 雖然有些安全專家承認微軟在Vista上工作出色，但他們也提醒人們：不要希望UAC能在一夜間消滅與管理員帳號有關的所有問題。<ref name = "ZOL"/> 此外，使用者帳戶控制對使用者的某些要求也飽受批評。UAC的確認對話框時常出現就讓許多用戶感到厭煩，而且在某種程度上讓部分人士覺得有些莫名其妙。導致許多使用者很容易因為厭煩而乾脆關掉UAC這項功能，或者根本不去理解UAC警報訊息的具體內容，直接點擊「繼續」按鈕。雖然使用者帳戶控制也可以透過控制面板被關閉，但是這也會禁止一些藉由使用權限分離所運行的功能，如[[Internet_Explorer_7|Internet Explorer 7]]的安全瀏覽模式；這是因為Internet Explorer 7這模式主要是依賴UAC的運作，把有安全風險內容的文件檔案，先把其虛擬化後再恢復到原來的樣子。

「Vista的確提供了很多功能來保護你的系統，但前提是要恰當地使用它。」微軟在Vista的最佳實踐指南中談到UAC時有這樣一段話：「系統到底有多安全取決於使用者的操作，因此在點擊按鈕時最好想一下。」實際上，微軟並不鼓勵用戶把UAC當成一個安全的堡壘。也因此，微軟公司大都也不認為UAC於現實上的漏洞即為安全漏洞。<ref name = "ZOL"/>

=== Windows防火牆 ===
自從[[Windows_XP|Windows XP]]推出SP2以來，Windows系統在安裝時其[[防火牆_(計算機)|防火牆]]功能在預設下是開啟的。而在[[Windows_Vista|Windows Vista]]中，Windows防火牆除了像XP中一樣能過濾由外界到電腦的封包，還能過濾由電腦到外界的封包。這能防範日益增加的[[間諜軟件|間諜軟件]]、[[網絡釣魚|網絡釣魚]]以及[[分散式拒絕服務攻擊|分散式拒絕服務攻擊]]的威脅。<ref name = "people"/> 然而Vista在安裝時，對向外發送的封包進行過濾這一功能在預設下是關閉的。換句話說，如果不另外進行手動設定，Vista的防火牆對網路連接所提供的保護，與XP SP2相比沒有多大區別。<ref name = "ZOL"/>

=== Windows Defender ===
[[Windows_Defender|Windows Defender]]系統提供了防[[間諜軟體|間諜軟體]]的功能。<ref name = "people"/> 但是根據間諜軟體供應商[[Webroot|Webroot]]的說法，Windows Defender對絕大多數的間諜軟件都不起作用。2009年2月，有[[駭客|駭客]]發現Windows Defender有可能成為惡意軟體攻擊Vista的跳板。因為微軟的[[惡意軟體|惡意軟體]]偵測引擎有一個漏洞，可以被惡意者發起對Vista的攻擊。<ref name = "ZOL"/>

未經登記而能安裝的驅動程式，在預設中僅包括Vista上的初期使用工具，以及如Atsiv等部分第三方工具。<ref>{{Cite web |url= http://www.computerworld.com/s/article/9028440/Utility_evades_Vista_kernel_defenses |title= Utility evades Vista kernel defenses ─ Aussie software can get around 64-bit Vista's signed-code requiremen |author= Gregg Keizer |date= 2007年7月30日 |publisher= [[Computerworld|Computerworld]]  |language= en |accessdate= 2010年12月18日 |quote=  }}</ref> 但微軟Service Pack 1中，已藉由修復程序KB932596修改這設定。<ref>{{Cite web |url= http://support.microsoft.com/kb/932596 |title= 若要改善核心補充程式保護更新 Microsoft 安全性摘要報告： |author=  |date= 2009年9月17日 |publisher= [[微軟|微軟]]  |language= zh-hant |accessdate= 2010年12月18日 |quote=  }}</ref> 雖然微軟聲稱藉由採用嚴格的驅動程序審查，意味著將會有更多的安全性，但一些{{谁|批評者}}認為現在幾乎很少有任何安全攻擊，是直接且蓄意的放在軟體驅動程式上，且大多數的驅動程式幾乎是由設備製造商自行編寫出的。一些使用[[UltraDefrag|UltraDefrag]]開發人員亦也抱怨此功能，因為這會防止他們的所需的「ultradfg.sys」載入。

=== 內建保護功能的缺陷 ===

=== 應用程序的Bug ===
儘管為了減少應用程序中最常見的漏洞，給整個系統帶來危害，Vista在這方面做了很多工作，如：採用內存地址空間的隱藏、對作業系統內核的保護等技術，導致駭客想利用普通的Bug來發起對系統的總攻擊將非常困難。而[[賽門鐵克|賽門鐵克]]進行的初步研究也證明，儘管Vista在某些形式的攻擊面前仍然比較脆弱，不過Vista基本上仍有達到了[[微軟|微軟]]預先設想的安全目標。

但在2009年1月，有[[駭客|駭客]]發現可以透過Vista的[[語音識別|語音識別]]功能訪問遠程系統，進而執行包括刪除系統中的任意文件等功能。與之前的各種[[Windows|Windows]][[作業系統|作業系統]]版本一樣，針對運行Vista系統的電腦的攻擊主要形式不是利用作業系統本身，而是利用了運行在作業系統之上的各種應用系統的漏洞，加以進攻。<ref name = "ZOL"/>

此外，雖然Vista把[[.Net|.Net]]作為首選的開發[[系統平台|平台]]，在Net開發平台的[[託管代碼|託管代碼]]和安全沙漏功能下，有助於幫助開發人員避免犯一些常見且可能成為駭客利用的程序錯誤。但如此一來開發人員就必須重寫原來的代碼，而那些以前開發的應用程序則無法從Vista受益，它們仍然還像從前一樣容易被駭客們利用其中的漏洞。

== 數位著作權管理 ==
{{main|自由軟體基金會反Windows運動}}
[[File:VistaParty3-cropped.jpg|right]][[波士頓|波士頓]]的「[[自由軟體基金會反Windows運動|BadVista]]」遊行運動。]]

另一種常見的批評是於作業系統中、關於已規格化的[[數位著作權管理|數位著作權管理]]（DRM），尤其是在[[受保护媒体路径|影像檔案路徑]]（PVP）特別受到限制，其中亦涉及到了[[HDCP|HDCP]]和[[ICT|ICT]]等技術。這功能被添加到Vista的原因主要是因為[[微軟|微軟]]與[[好萊塢|好萊塢]]工作室彼此雙方的協議所致。<ref name="microsoft">{{Cite web|url=http://www.microsoft.com/whdc/device/stream/output_protect.mspx |title=Output Content Protection and Windows Vista |author=Ed Bott |work=[[網際網路檔案館|網際網路檔案館]] |date=2005年4月27日 |publisher=[[微軟|微軟]] |language=en |accessdate=2010年12月19日 |quote= |deadurl=bot: unknown |archiveurl=https://web.archive.org/web/20061116163834/http://www.microsoft.com/whdc/device/stream/output_protect.mspx |archivedate=2006年11月16日 }}</ref> 除了有關軟體的其他事項外，這協議也將涉及[[HD_DVD|HD DVD]]和[[藍光光碟|藍光光碟]]裡面需保護版權的內容，但實際上在2010年前仍未有具體限制。

當用者藉由傳播影片的軟體傳送指定內容時，可能會因此出現受「保護」的提示，更甚者可能會被攔截。此外，所有與協議相關的內容時，必須通過微軟認證方能使用。<ref name = "microsoft"/> 在開始播放之前，所涉及使用的設備其硬體功能將會被掃描檢查，驗證是否為正版，且並沒有被篡改過；而不支援HDCP保護者，其介面將會被關閉或者蓄意降低任何訊號輸出效率。此外，由於微軟針對全球的指定設備的清單已遭破壞，目前這份名單主要是透過[[網際網路|網際網路]]將清單內容傳至個人電腦中，並於網路上擁有更新機制。這樣的唯一影響為撤銷部分協議的限定功能，除了高級別的指定內容仍不能使用外，其他功能包括低清晰度轉放等則持續保留。<ref name = "microsoft"/><ref name = "Answers">{{Cite web |url= http://windowsteamblog.com/windows/archive/b/windowsvista/archive/2007/01/20/windows-vista-content-protection-twenty-questions-and-answers.aspx |title= Windows Vista Content Protection - Twenty Questions (and Answers) |author= Nick White |date= 2007年1月20日 |publisher= Windows Vista Team Blog |language= en |accessdate= 2010年12月19日 |quote=  }}</ref>

[[自由軟體基金會|自由軟體基金會]]曾因此對Vista進行一項名為「BadVista」[[自由軟體基金會反Windows運動|遊行運動]]。

=== 值得注意的批評 ===
[[紐西蘭|紐西蘭]][[奧克蘭大學|奧克蘭大學]]電腦安全專家Peter Gutmann在其發行的[[白皮書|白皮書]]中，對這機制提出了以下幾個問題：<ref>{{Cite web |url= http://www.cs.auckland.ac.nz/~pgut001/pubs/vista_cost.html |title= A Cost Analysis of Windows Vista Content Protection |author= Peter Gutmann |date= 2007年6月12日 |publisher=  |language= en |accessdate= 2010年12月19日 |quote=  }}</ref><ref>{{Cite web |url= http://max-sievers.name/vista_cost.pdf |title= A Cost Analysis of Windows Vista Content Protection |author= Peter Gutmann |date= 2007年1月27日 |publisher=  |language= en |accessdate= 2010年12月19日 |quote=  |archive-url= https://web.archive.org/web/20110511144644/http://max-sievers.name/vista_cost.pdf |archive-date= 2011年5月11日 |dead-url= yes }}</ref>

* 如果增加加密措施，將使介面變得更加昂貴，而其成本也會反應在使用戶購買的價錢上。
* 如果輸出的介面並不具支援時，那麼依此即使是不錯的介面也可能被要求關閉（如[[S/PDIF|S/PDIF]]）。
* 因為製造商可能自行另有要求，一些較新的高清晰度顯示器可能並不支持[[HDCP|HDCP]]功能。
* 新增加的功能由於其複雜性，使系統變得更加不可靠。
* 由於部分非指定的媒體並沒有受到限制，可能會因此變相鼓勵用戶改變選擇，來觀看這些沒有限制的內容，進而擊潰這計劃的最初目的。
* 保護機制仍有可能會出現錯誤訊息，導致使用戶漸漸拒絕此服務。
* 取消認證的產品廣泛是受使用戶好評的的軟體，Peter Gutmann懷疑[[微軟|微軟]]是否有成為有史以來，第一位將此計劃徹底實現的能力。但另一方面，他們也可能因為受與電影工作室的法律義務，仍被迫持續落實。

=== 對批評的反應 ===
[[Windows_Vista|Windows Vista]]軟體編輯者[[Ed_Bott|Ed Bott]]，在[[部落格|部落格]]上寫了三篇文章，駁斥了許多Gutmann的說法。<ref>{{Cite web |url= http://www.zdnet.com/blog/bott/everything-youve-read-about-vista-drm-is-wrong-part-1/299 |title= Everything you've read about Vista DRM is wrong (Part 1) |author= Ed Bott |date= 2007年9月16日 |publisher= [[ZDNet|ZDNet]] |language= en |accessdate= 2010年12月19日 |quote=  }}</ref> Ed Bott的批評可以概括如下：

* Gutmann的論文資料取自微軟過時的文件和二手網路資料。
* Gutmann選擇性地採納微軟公布的規格。
* Gutmann並沒有使用Vista來證明他的理論。相反地，他先做出錯誤的假設，然後再瘋狂地推測其影響。
* Gutmann的文件表面上為一認真的研究，實際上頂多只能算是一般意見而已。

科技作家[[George_Ou|George Ou]]亦認為Gutmann的文件依賴不可靠的來源，且從未使用過Windows Vista來測試他的理論。<ref>{{Cite web |url= http://www.zdnet.com/blog/ou/gutmann-vista-drm-paper-uses-shoddy-web-forums-as-source/723 |title= Gutmann Vista DRM paper uses shoddy Web Forums as source |author= George Ou |date= 2007年9月1日 |publisher= [[ZDNet|ZDNet]] |language= en |accessdate= 2010年12月19日 |quote=  }}</ref> 此外，微軟已在其部落格上發布了《二十個問題（及答案）》和其他Windows Vista內容保護的相關文章，準備反駁部分Gutmann的論點。<ref name = "Answers"/> 微軟還指出，內容保護機制早在Windows版本為[[Windows_Me|Windows Me]]時就已經存在了。<ref name = "Answers"/>

Gutmann在回應Bott和Ou雙方的文章中提到，其文章並沒有因此被駁倒，並反指Bott所著的三篇文章為「謠言」。<ref>{{Cite web |url= http://www.cypherpunks.to/~peter/zdnet.html |title= Windows DRM: A Response to the Disinformation |author= Peter Gutmann |date=  |publisher=  |language= en |accessdate= 2010年12月19日 |quote=  |archive-url= https://web.archive.org/web/20110607140702/http://www.cypherpunks.to/~peter/zdnet.html |archive-date= 2011年6月7日 |dead-url= yes }}</ref> 而[[微軟最有價值專家|微軟最有價值專家]][[Paul_Smith|Paul Smith]]在回應Gutmann的文件中提到了他的一些論點，其中包括了：<ref>{{Cite web |url= http://www.dasmirnov.net/blog/windows_vista_drm_nonsense |title= Windows Vista DRM nonsense |author= Paul Smith |date= 2006年12月31日 |publisher=  |language= en |accessdate= 2010年12月19日 |quote=  }}</ref>

* 影像檔案路徑將不會被立刻限制，並將會持續有相當一段時間。有人說這是微軟和[[索尼|索尼]]之間的另一項協議，至少到2010年為止藍光光碟並不會被管制，甚至可能直到2012年。<ref>{{Cite web |url= http://arstechnica.com/old/content/2006/05/6880.ars |title= Hollywood reportedly in agreement to delay forced quality downgrades for Blu-ray, HD DVD |author= Ken Fisher |date= 2006年5月21日 |publisher= Arstechnica.com |language= en |accessdate= 2010年12月19日 |quote=  }}</ref>

== 軟體保護平台 ==
Vista接續Windows XP的[[Windows正版增值計劃|Windows正版增值計劃]]擁有強化的防拷貝技術，這被稱為軟體保護平台（SPP）。<ref>{{Cite web |url= http://www.computerworld.com/s/article/9004970/The_Skinny_on_Windows_SPP_and_Reduced_Functionality_in_Vista |title= The Skinny on Windows SPP and Reduced Functionality in Vista |author= Scot Finnie |date= 2006年11月10日 |publisher= [[Computerworld|Computerworld]]  |language= en |accessdate= 2010年12月19日 |quote=  }}</ref> 在最初的版本的Windows Vista（無下載Service Pack 1）中這是一時常更新失敗的功能，在更新時常會出現「失敗的[[產品激活|產品激活]]」，或者誤認使用戶的版本「為假冒或非正版」。<ref>{{Cite web|url=http://www.microsoft.com/presspass/features/2006/oct06/10-04SoftwareProtection.mspx |title=Microsoft’s Software Protection Platform: Protecting Software and Customers from Counterfeiters |author=Cori Hartje |date=2006年10月4日 |publisher=[[微軟|微軟]] |language=en |accessdate=2010年12月19日 |quote= |deadurl=yes |archiveurl=https://web.archive.org/web/20070706203635/http://www.microsoft.com/presspass/features/2006/oct06/10-04SoftwareProtection.mspx |archivedate=2007年7月6日 }}</ref> Microsoft於白皮書描述說：

「如果默認的[[網頁瀏覽器|網頁瀏覽器]]被啟動時，用戶將看到一個購買新產品密鑰的選項。同時會沒有開始列表、桌面圖標，且[[桌面背景|桌面背景]]將會變為黑色......一小時後，系統便會再次提出警告。」

這被批評為過於嚴苛，<ref>{{Cite web |url= http://www.computerworld.com/s/article/9004916/Hands_On_A_Hard_Look_at_Windows_Vista?pageNumber=15#spp |title= Hands On: A Hard Look at Windows Vista |author= Scot Finnie和Preston Gralla |date= 2006年11月10日 |publisher=  [[Computerworld|Computerworld]] |language= en |accessdate= 2010年12月19日 |quote=  }}</ref><ref>{{Cite web |url= http://www.zdnet.com/blog/bott/for-vista-wga-gets-tougher/148 |title= For Vista, WGA gets tougher |author= Ed Bott |date= 2006年10月4日 |publisher=  [[ZDNet|ZDNet]] |language= en |accessdate= 2010年12月19日 |quote=  }}</ref> 特別是考慮到系統錯誤時的誤判以及臨時性驗證服務停止時的誤認。<ref>{{Cite web |url= http://www.zdnet.com/blog/bott/another-wave-of-wga-failures/150 |title= Another wave of WGA failures |author= Ed Bott |date= 2006年10月4日 |publisher=  [[ZDNet|ZDNet]] |language= en |accessdate= 2010年12月19日 |quote=  }}</ref> 有許多報導指出使用戶在將正版XP升級至Vista時，以及在執行Windows更新時，常被判斷為「非正版」。<ref>{{Cite web |url= http://arstechnica.com/microsoft/news/2007/08/windows-genuine-advantage-suffers-worldwide-outage-problems-galore.ars |title= Windows Genuine Advantage suffers worldwide outage, problems galore (updated) |author= Ken Fisher |date= 2007年8月25日 |publisher= ArsTechnica |language= en |accessdate= 2010年12月19日 |quote=  }}</ref><ref>{{Cite web |url= http://blogs.msdn.com/b/wga/archive/2007/08/27/update-on-validation-issues.aspx |title= Update on Validation Issues |author= MSDNArchive  |date= 2007年8月27日 |publisher= [[微軟開發者網路|微軟開發者網路]] |language= en |accessdate= 2010年12月19日 |quote=  }}</ref>

== 與新舊版本比較 ==
目前網路及媒體上的主流論點大都認為[[Windows_7|Windows 7]]比[[Windows_Vista|Windows Vista]]好許多；<ref>{{Cite web |url= http://www.pctartarus.com/?bbsthread-520/ |title= 微軟對 Windows 7 , Vista 及 XP 的取捨 |author= pctartarusadmin |date= 2009年1月13日 |publisher= PCTartarus 地獄電腦 |language= zh-hant |accessdate= 2010年12月19日 |quote=  }}{{dead link|date=2017年12月 |bot=InternetArchiveBot |fix-attempted=yes }}</ref> 而在Windows 7正式版上市前，許多使用者在購買電腦時也有案例要求店家將Windows Vista降級為舊版本的[[Windows_XP|Windows XP]]。而在早些2007年時，[[微軟|微軟]]還是允許電腦生產商向雖然購買了Vista系統電腦，但還想使用XP系統的消費者，提供系統「降級」的服務。雖然這項服務只適用於Windows Vista企業版和旗艦版，而如何進行則是由電腦製造商來決定。其中[[富士通|富士通]]為該計畫最積極的支持者之一，在2007年8月時已經開始在[[筆記型電腦|筆記型電腦]]包裝盒裡放入了XP光碟；富士通公司的營銷經理[[布蘭登·法里斯|布蘭登·法里斯]]稱，這對中小企業非常有好處。

[[惠普|惠普]]也於同時亦啟動一項針對大多數商務電腦型號的計畫，惠普公司女發言人[[蒂凡妮·史密斯|蒂凡妮·史密斯]]表示，對於商務桌上型電腦、工作站和某些商務筆記型電腦的用戶，他們在配置系統時可在支付很少費用或免費的狀況下獲得一張XP專業版的恢復光碟。她並認為現在評估客戶的興趣有大為時過早，由於Windows Vista剛推行一個月，並無法真正了解市場的需求。 對於批評Vista的觀點，也有一些電腦商戶則對Vista表示寬容態度，認為「當初的XP推出時，也曾有人批評XP是『垃圾』」。

2008年6月30日，微軟宣布終止出售XP授權及零售版本，不過當時市場上仍濔漫著如何取得XP的討論（例如購買安裝[[Windows_Vista_Business|Windows Vista Business]]及[[Windows_Vista_Ultimate|Windows Vista Ultimate]]電腦的客戶可要求降級到Windows XP Professional，以及低價電腦上微軟仍會提供低階的XP版本）。<ref name = "Steven Ballmer"/>

2009年10月Windows 7推出後，一些專家們間接的稱讚說：「Windows 7作為微軟的下一代作業系統，它已經具備了那些Windows Vista本應具有而未被設計的優點。」 一位已經使用了Windows 7試用版幾個月的[[加利福尼亞州|加利福尼亞州]]技術顧問Rob Enderle評論說：「Windows 7被設計成用來修補Windows Vista中存在的主要問題。這是自Windows Vista推出以來，在不改變基本計算機代碼前提下所能做出的最大改變。」「Windows Vista被設計成比Windows XP具有更好的[[網絡安全|網絡安全]]性，而這正是微軟想要保留的。因此Windows 7在此基礎上做了改進，保留了Windows Vista的網路安全性的優點。」

== 微軟反應 ==
[[微軟|微軟]]於2008年開始推動[[Windows_Vista|Windows Vista]]的形象改良計畫，內部代稱為「FTP」。內容除了花費資金在《[[華爾街日報|華爾街日報]]》及《[[今日美國|今日美國]]》上發布眾多廣告之外，也有如加強對小企業宣傳，以及於同年7月上旬提供用戶免費Vista相關技術支援的「 Assurance」方案。<ref name="Mojave">{{Cite web |url= http://www.zdnet.com.tw/news/software/0,2000085678,20130743,00.htm |title= 微軟用'Mojave'洗刷Vista的形象 |author=  |date= 2008年7月25日 |publisher= [[ZDNet|ZDNet]] |language= zh-hant |accessdate= 2010年12月19日 |quote=  |archive-url= https://web.archive.org/web/20100305121633/http://www.zdnet.com.tw/news/software/0,2000085678,20130743,00.htm |archive-date= 2010-03-05 |dead-url= yes }}</ref> 此外，亦利用Vista批評者對Vista的改變印象來強力推銷，如2008年7月時微軟於[[舊金山|舊金山]]，將多名Vista印象不好的Windows XP用戶召集，讓他們觀賞一支介紹Vista的錄影帶，在詢問其對Vista的印象後，再播放另一支影帶介紹代號「Mojave」的「新」[[作業系統|作業系統]]。結果發現，逾九成的受訪者對「新」作業系統的印象良好，但微軟事後告訴受訪者其實「Mojave」就是Windows Vista。<ref name="Veghte">{{Cite web |url= http://www.zdnet.com.tw/news/software/0,2000085678,20130745,00.htm |title= 微軟：Windows 7進度如常 |author= Ina Fried |date= 2008年7月25日 |publisher= [[ZDNet|ZDNet]] |language= zh-hant |accessdate= 2010年12月19日 |quote=  |archive-url= https://web.archive.org/web/20081101123137/http://www.zdnet.com.tw/news/software/0,2000085678,20130745,00.htm |archive-date= 2008-11-01 |dead-url= yes }}</ref>

Windows事業部負責人[[Bill_Veghte|Bill Veghte]]對此事在接受[[CNET|CNET]]訪問時表示他希望嘗試新的方法為：讓訊息傳遞出去。Veghte指出除了早期碰上的技術挑戰外，Vista遭遇的最主要是感受問題。<ref name = "Mojave"/><ref name = "Veghte"/> 而造成民眾對Vista反感的理由之一，可能是受[[蘋果公司|蘋果公司]]強打反Vista廣告宣傳的影響。而微軟總裁[[史蒂夫·巴爾默|史蒂夫·巴爾默]]也在其備忘錄中對員工表示：「未來數周，我們將發動一場攻勢，消除顧客對Windows Vista殘餘的疑慮。今年稍後，還會有更全面性的行動，以重新界定Windows對顧客的意義與價值。」

但早在2008年7月9日，微軟負責Vista消費市場的副總裁[[布拉德·布魯克斯|布拉德·布魯克斯]]在微軟全球合作夥伴年會上就已公開承認Vista存在問題，他公開表示：「我們已經打破了很多東西，這給大家帶來不適，消費者可能會想，Vista是否值得我們關注。」他也提到了蘋果廣告對Vista的批評「我們的競爭對手一直在鼓噪。......大家都知道，這帶來了一定影響。對此我們將予以反擊，我們認為，他們（蘋果）的說法傳遞了一個悲觀的訊息，事實上軟體必須融入日常生活的各個方面。」而2010年5月21日，微軟總裁[[史蒂夫·巴爾默|史蒂夫·巴爾默]]在微軟年度CEO高峰會的一個主題演講中坦承：「Vista做得不是很好，這說的不是該產品本身，而是微軟曾在五、六年時間中沒有一個新產品推出。」他認為微軟推出Windows新版本等待的時間過長，而且在Vista上做得過多。

長期研究微軟的市場研究公司[[Directions|Directions]]的分析師[[邁克爾·切瑞|邁克爾·切瑞]]表示，Windows Vista並不像人們想像的那麼糟糕。而微軟一直秉承的「沉默是金」的策略，事實上卻是弊大於利。 他說：「微軟最大的失誤在於沒有充分利用今年2月份發布的Vista SP1這一機會，來向用戶展示Vista是一款穩定且值得信賴的系統，尤其是現在很多設備製造商已經更新了自己的驅動程序。」切瑞認為，微軟本可以藉此向用戶說明，Vista在安全上的改進是非常重要的，這樣不僅可以對[[使用者帳戶控制|使用者帳戶控制]]（UAC）和BitLocker等安全功能進行推廣，而且能夠讓更多的用戶選購能夠更好利用Vista SP1的產品，但是微軟卻錯失了這一機會。
