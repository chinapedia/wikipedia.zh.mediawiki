'''用-{}-户'''，又称'''使-{}-用者'''，是指使用[[电脑|电脑]]或[[计算机网络|网络]][[服务|服务]]的人，通常拥有一个'''用户账号'''，并以'''用户名'''识别。用户有时也泛指没有充分技术经验的人。<ref>Jargon File entry for {{cite web |url=http://catb.org/jargon/html/U/user.html |title=User |accessdate=November 7, 2010}}</ref>

==终端用户==
{{main|终端用户}}

[[终端用户|终端用户]]既指软件的最终操作者，也是[[软件工程|软件工程]]内的一个概念，指终端用户的抽象集合，用于区分单纯使用软件的使用者和进行[[软件开发|软件开发]]的开发者。这种抽象主要在设计[[用户界面|用户界面]]时有用，用于代表普通用户的共同特性。在[[以用户为中心的设计|以用户为中心的设计]]中通常会创建多个[[人物誌_(用戶體驗)|人物誌]]以代表典型的最终用户，每个人物誌都可能包括对不同界面的接受程度，以及在某一专业领域的知识。

虽然终端用户类别没有限制，在[[软件设计|软件设计]]时通常会假设终端用户拥有极少的技术知识和训练，特别是在为普通大众设计程序时。<ref>{{cite web|author1=Vangie Beal|title=What is End User? Webopedia Definition|url=http://www.webopedia.com/TERM/E/end_user.html|website=www.webopedia.com|publisher=webopedia|accessdate=2017-05-22|language=en}}</ref>在这个环境下，为了增加[[易用性|易用性]]，通常倾向于实现易于学习的[[图形用户界面|图形用户界面]]（可能有[[触摸屏|触摸屏]]），而非复杂的[[命令行界面|命令行界面]]。

{{link-en|终端用户开发|end-user development}}的概念模糊了通常的用户和开发者之间的界限，给予非专业人士在不了解[[编程语言|编程语言]]的情况下创造复杂对象的能力。

==用户账号==
用户可以通过账号向系统服务进行[[身份验证|身份验证]]，并取得相关授权，但身份验证并不代表授权。为了[[会计学|会计]]、[[计算机安全|安全]]、数据收集和资源管理，用户通常需要一个[[密码|密码]]或其他证明以登录账号。当用户登录后，操作系统通常会用识别号标识用户，而不是用户名。

电脑系统可根据用户数量分为两类：
* 单用户系统，不存在多个账户的概念
* [[多用户|多用户]]系统，存在多个账户的概念，要求用户在使用系统之前识别身份。

多用户系统上的用户通常有一个[[家目录|家目录]]，其储存与用户相关的[[電腦檔案|文件]]，其他用户无法接触。用户账号通常包含一份公开的用户信息，包含由用户提供的基本资料。

==网名==
在[[网络社区|网络社区]]，网名可以成为人物的标志，其中一些通过其持有人的贡献获得了一定水平的名气。一些著名的网名：

*CmdrTaco，持有者{{link-en|罗伯·马尔达|Rob Malda}}，资讯科技网站[[Slashdot|Slashdot]]的创始人

有时网名只是实名的缩写或字首。在这种情况下，持有人的名气与网名的名气很难分开。类似的例子有：

*Matz，持有者[[松本行弘|松本行弘]]，编程语言[[Ruby|Ruby]]的设计者和实现者
*RMS，持有者[[理查德·斯托曼|理查德·斯托曼]]，[[GNU通用公共许可证|GNU通用公共许可证]]作者，[[自由软件基金会|自由软件基金会]]创始人，[[Emacs|Emacs]]首席维护员
*JWZ，持有者[[杰米·加文斯基|杰米·加文斯基]]（Jamie Zawinski），[[Mozilla|Mozilla]]、[[网景导航者|网景导航者]]、[[XEmacs|XEmacs]]和{{link-en|Xscreensaver|Xscreensaver}}的作者/合作者
*ESR，持有者[[埃里克·雷蒙|埃里克·雷蒙]]，[[开放源代码促进会|开放源代码促进会]]创始人之一,[[大教堂和市集|大教堂和市集]]的作者
*billg，持有者[[比尔·盖茨|比尔·盖茨]]，[[微软|微软]]创始人之一
*Jimbo Wales，持有者[[吉米·威尔士|吉米·威尔士]]，[[维基百科|维基百科]]创始人之一
