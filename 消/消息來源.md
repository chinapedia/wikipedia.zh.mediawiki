'''消息來源'''<ref>'''消息來源'''是[[Mozilla_Firefox|Mozilla Firefox]]正體中文版中的譯名。</ref>（{{lang-en|'''web feed'''、'''news feed'''、'''syndicated feed'''}}又譯為'''源料'''<ref>[http://www.taipei.gov.tw/taipei_news/2005/theme/06_1102.htm 邁入新世紀─從部落格、維基百科與源料開始] {{webarchive|url=https://web.archive.org/web/20080926130709/http://www.taipei.gov.tw/taipei_news/2005/theme/06_1102.htm |date=2008-09-26 }}</ref>、'''饋送'''<ref>[http://mail.google.com/support/bin/answer.py?answer=18218&hl=cn 什么是 RSS 和 Atom 馈送？]</ref>、'''資訊提供'''<ref>[http://mail.google.com/support/bin/answer.py?answer=18218&hl=b5 何謂 RSS 和 Atom 資訊提供？]</ref>、'''供稿'''<ref>[http://www.google.com/support/reader/bin/answer.py?answer=69967&hl=cn 什么是供稿？订阅是什么意思？]</ref>、'''摘要'''、'''源'''<ref>'''摘要'''是[[Internet_Explorer|Internet Explorer]]繁體中文版中的譯名；'''源'''是Internet Explorer簡體中文版中的譯名；</ref>、'''新闻订阅'''、'''网源'''<ref>'''新闻订阅'''是[[Opera|Opera]]正体中文版的中的译名；'''网源'''是Opera简体中文版中的译名；</ref>）是一種[[文件格式|資料格式]]，[[網站|網站]]可透過它將最新資訊傳播給用戶，用戶能夠訂閱網站的先決條件是網站可提供持續更新的資訊。消息來源受到[[網誌|網誌]]及[[新聞|新聞]]網站的廣泛採用，因為此類型的網站經常更新內容。如前所述，feed的譯名很多，莫衷一是，至2008年底為止，還沒有一個十分通用而備受認可的中文譯名。將feed匯流於一處稱為聚合（aggregation），而用於聚合的[[軟體|軟體]]稱為[[聚合器|聚合器]]（aggregator）。對終端使用者而言，聚合器是專門用來訂閱網站的軟體，一般亦稱為[[RSS閱讀器|RSS閱讀器]]、feed閱讀器、新聞閱讀器等。

典型的一種應用是，網站提供feed[[URL|網址]]（通常以橘色的feed[[標識|標識]]供用戶點擊，如[[File:Feed-icon.svg|16px]]或[[File:RSS_icon.svg|File:RSS icon.svg]]），而用戶將feed網址登記到閱讀器裡，就可以訂閱此網站的最新資訊。閱讀器操作方式類似[[電子郵件客戶端|電子郵件客戶端]]軟體，只是用feed閱讀器訂閱網站不需電子郵件位址。用戶訂閱了之後，閱讀器便會透過網站feed檢查網站有無更新。視閱讀器的功能而定，用戶可以得知哪些訂閱的網站更新了，或得到已訂閱網站的新資訊列表，或自動下載已訂閱網站的新內容。閱讀器可以設定[[排程|排程]]，定時地檢查已訂閱的所有網站。網站feed雖然看似將內容[[推送技术|-{zh-hant:推播;zh-hans:推送}-]]給用戶，但事實上它采用的是拉取技术（pull technology）。

常用的feed格式有[[RSS|RSS]]、[[Atom_(標準)|Atom]]兩種。由於RSS是較早通行的格式，一些網站雖然提供了兩種格式，但只稱為RSS。feed傳播給用戶的內容種類通常是[[HTML|HTML]]（網頁內容）或通往網頁的鏈結。許多網站只在feed當中放入文章的一部分或著摘要，想看全文就得要點擊鏈結到原站去看；然而，也有一些人認為應該將全文放入feed<ref>{{cite web |url=http://blog.donews.com/keso/archive/2006/01/23/705083.aspx  |title=东拉西扯：全文还是摘要 |accessdate=2008-11-23 |author=洪波（keso）}}</ref><ref>{{cite web |url=http://www.chunmin.com/archives/71 |title=從 Google Gears 談全文 RSS |accessdate=2008-11-23 |author=常純敏 |archive-url=https://web.archive.org/web/20080905145721/http://www.chunmin.com/archives/71 |archive-date=2008-09-05 |dead-url=yes }}</ref>。除了網頁之外，feed內容也可以是通往其他[[數碼|數位]][[多媒體|多媒體]]的鏈結。供用戶訂閱的內容若是[[音訊|音訊]]或[[視訊|視訊]]等多媒體資訊，則稱為[[播客|播客]]（podcast）。

== 閱讀器 ==
{{main|聚合器}}
就像[[電子郵件|電子郵件]]一樣，feed閱讀器除了有[[客戶端|客戶端]]軟體（如[[FeedDemon|FeedDemon]]、[[NetNewsWire|NetNewsWire]]），亦有數種線上閱讀器（如[[Bloglines|Bloglines]]、[[Google閱讀器|Google閱讀器]]、[[funP|funP]]哈部落、[[抓虾|抓虾]]等）。由於網誌的盛行，當代[[瀏覽器|瀏覽器]]當中，[[Internet_Explorer|Internet Explorer]]（第7版起<ref>[http://www.microsoft.com/taiwan/windows/ie/default.mspx#ie7rss 讓新聞直接傳送給您。]</ref>）、[[Mozilla_firefox|Mozilla firefox]]、[[Safari|Safari]]、[[Opera|Opera]]等都整合了訂閱feed的功能。[[電子郵件客戶端|電子郵件客戶端]]軟體[[Windows_Live_Mail|Windows Live Mail]]、[[Mozilla_Thunderbird|Mozilla Thunderbird]]也提供了訂閱feed的功能。有些[[行動電話|行動電話]]能夠訂閱feed，如[[諾基亞|諾基亞]]的[[S60|S60]]系統<ref>{{en}}[http://press.nokia.com/PR/200511/1019239_5.html Nokia introduces a new Web browser for S60 3rd Edition] {{Wayback|url=http://press.nokia.com/PR/200511/1019239_5.html |date=20070423120911 }}</ref>。許多個人化首頁服務，如[[iGoogle|iGoogle]]、My Yahoo、My MSN等也使用了feed相關技術，這些服務通常預設了天氣、股市等幾種常用的feed資訊，但也允許用戶自行添加訂閱[[第三方|第三方]]的feed網址。

== 與電子郵件的比較 ==
[[郵件列表|電子郵件列表]]（有時稱為[[電子報|電子報]]）是另一種網站通知用戶新資訊的方式。在feed相關技術成熟之前，郵件列表佔有主導地位。閱讀器的運作原理有些類似於[[電子郵件|電子郵件]]的[[POP3|POP3]]通訊協定，只是feed閱讀器查看的並非用戶自己的郵箱，而是網站上的[[電腦檔案|文件]]，文件內含有此網站的更新資訊。和用電子郵件通知用戶的方式比較起來，feed有一些好處：

* 訂閱feed時用戶不必提供電子郵件位址，用戶從而免去了與電子郵件有關的風險與威脅，如[[垃圾郵件|垃圾郵件]]、[[電腦病毒|電腦病毒]]、[[釣魚式攻擊|釣魚式攻擊]]、[[尼日利亞騙徒|郵件詐騙]]等。
* 當用戶想要停止接收新資訊通知時，不必向發信者申請「退訂」，只要將feed網址從自己的閱讀器的已訂閱清單中刪除即可。
* 結構化的格式（除了某些[[RSS|RSS]]版本外，大都基於[[XML|XML]]），機器易於處理，有利於[[Web_2.0|Web 2.0]]式的[[混搭|混搭]]。

== 應用與發展 ==
「出版（publish）feed」及「供稿（syndication）」是兩個常用詞彙，用來描述網站為某種資訊提供feed。就像供稿給[[報紙|報紙]]的文章一樣，feed內容可以被其他網站重新出版。feed的格式被設計成[[機器可讀|機器可讀]]而非人類可讀的，這使得它可以[[自動|自動]]將資訊從一個網站傳遞到另一個網站，完全不必透過人類手動轉貼。其他網站可以利用feed聚合各種資訊，比如說將許多網誌匯流至同一處而成為[[部落格聯播網|部落格聯播網]]。一些網誌供應商（BSP）提供了在網誌側邊欄顯示聯播網的功能。[[軟體開發|軟體開發]]界常將部落格聯播網網站稱為Planet，源自於一個線上聚合器軟體的名字<ref>{{en}}[http://www.planetplanet.org/ Planet] {{webarchive|url=https://web.archive.org/web/20051029095046/http://www.planetplanet.org/ |date=2005-10-29 }}</ref>。

較新的格式[[Atom_(標準)|Atom]]還提供了用戶張貼到網站的[[通訊協定|通訊協定]]，使得feed除了讀取（從網站獲得資訊）之外還多了寫入（將資訊送到網站）的新功能。這令[[可寫網路|可寫網路]]（writable web）更加標準化，利於新應用服務的撰寫與實現。
