{{Vgname|盜王之王|en=Thief: The Dark Project}}是{{link-en|望远镜工作室|Looking Glass Studios}}为[[Eidos_Interactive|Eidos Interactive]]开发的[[Microsoft_Windows|Microsoft Windows]]游戏，也是第一款把明暗光和声音运用到游戏操作中、同時以[[第一人称游戏|第一人称]]视角進行的[[個人電腦|個人電腦]][[隐蔽类游戏|隐蔽类游戏]]，被Eidos稱為[[策略游戏|策略]]與[[动作游戏|动作]]结合的游戏<ref>[https://web.archive.org/web/20001102010259/http://www.eidosinteractive.com/games/info.html?gmid=34 Thief - The Dark Project | Eidosinteractive.com]</ref>。遊戲背景設定在帶有[[中世紀|中世紀]]色彩之[[蒸汽朋克|蒸汽朋克]]大都會「城市」（The City），主角{{link-en|加洛特|Garrett (character)}}（Garrett）在接受一秘密組織的訓練後成為盜賊進行各種任務。

遊戲獲得一致好評，亦是望遠鏡工作室商業上最成功的作品，其潛行系統影響了後期同類別的遊戲。续作為2000年《[[神偷II：金属时代|神偷II：金属时代]]》、2004年《{{link-en|神偷：致命暗影|Thief: Deadly Shadows}}》和2014年重启作《[[神偷_(2014年游戏)|神偷]]》。

== 系統 ==
[[File:Baffscrn1.jpg|left]]

遊戲採用第一人稱視角和[[3D|3D]]的環境，強調潛行和避免正面交鋒，主角的戰鬥能力和抗傷害能力因此並不高<ref name="thiefmanual">{{cite book |title=Thief: The Dark Project Manual |last=Hart |first=Dorian |publisher=[[Eidos_Interactive|Eidos Interactive]] |year=1998 }}</ref><ref name="gamepro">{{cite web |url=http://www.gamepro.com/article/reviews/766/thief/ |title=Thief |last=Olafson |first=Peter |date=2000-01-01 |work=[[GamePro|GamePro]] |accessdate=2009-07-24 |archiveurl=https://web.archive.org/web/20100218033226/http://www.gamepro.com/article/reviews/766/thief/ |archivedate=2010-02-18 |deadurl=yes }}</ref>。停留在黑暗的地點可避免引起敵人注意；遊戲[[抬頭顯示器|抬頭顯示器]]中一個測量計能顯示主角的可視程度<ref name="thiefmanual" />。主角可進行側身、蹲下、攀爬、游泳和奔跑等動作<ref name="thiefmanual" />。遊戲內不同物件能產生不同程度的聲響，例如地毯較安靜而陶磚較嘈吵<ref name="retrospective">{{cite web |url=http://www.eurogamer.net/articles/retrospective-thief-the-dark-project-article |title=Retrospective: ''Thief The Dark Project'' |last=Walker |first=John |date=2009-05-17 |accessdate=2009-07-24 }}</ref>。主角可透過聲音從[[非玩家角色|非玩家角色]]接收訊息<ref name="leonard1" />，例如他們在甚麼地面上走動、與主角之間的距離及警戒程度<ref name="thiefmanual" />，也可利用聲音（如投擲物品）干擾非玩家角色。

遊戲設有十二個[[關卡_(電子遊戲)|關卡]]，各有不同任務目標，例如偷取指定的物品或累積贓物達一定價值，各目標均須完成才能前往下一個關卡；遊戲的三個難度對任務目標、關卡內的路徑及其他項目有所影響<ref name="leonard1" />。大部分關卡均不預設完成方式<ref name="leonard1">{{Cite journal |url=http://www.gamasutra.com/view/feature/131762/postmortem_thief_the_dark_project.php |title=Postmortem: ''Thief: The Dark Project'' |last=Leonard |first=Tom |work={{tsl|en|Game Developer Magazine}} |publisher=[[Gamasutra|Gamasutra]] |date=1999-07-09 |accessdate=2012-04-13 |pages=1–4 |issn=1073-922X |oclc=29558874 }}</ref>，容許玩家即興地進行遊戲<ref name="gamesnothollywood">{{cite web |url=http://www.gamasutra.com/view/feature/4032/beyond_pacing_games_arent_.php |title=Beyond Pacing: Games Aren't Hollywood |last=Wesolowski |first=Jacek |work=[[Gamasutra|Gamasutra]] |date=2009-05-21 |accessdate=2009-07-28}}</ref>。

遊戲內非玩家角色的[[人工智能|人工智能]]可使他們發現環境內的視覺和聽覺提示<ref name="leonard2">{{cite web |url=http://www.gamasutra.com/view/feature/2888/building_an_ai_sensory_system_.php?page=1 |title=Building an AI Sensory System: Examining the Design of ''Thief: The Dark Project'' |last=Leonard |first=Tom |date=March 7, 2003b |work=[[Gamasutra|Gamasutra]] |accessdate=2009-07-28 }}</ref>；假如他們看見主角、聽見主角發出的聲音、或察覺環境內出現血跡、被打開了的門及倒下的同伴等改變後會感到可疑，並視乎警戒狀態而無視提示或自動搜索主角<ref name="thiefmanual" /><ref name="leonard1" /><ref name="leonard2" />。非玩家角色主要分作守衛和僕人，前者會發出警告和攻擊主角，負傷達一定程度時會逃走<ref name="gamepro" />；後者則只會逃走並呼叫求救<ref name="allgame">{{cite web |url=http://www.allgame.com/game.php?id=11892&tab=review |title=''Thief: The Dark Project'' review |last=Suciu |first=Peter |work=[[Allgame|Allgame]] |accessdate=2009-08-12}}</ref>。非玩家角色也包括[[不死生物|不死生物]]，部分關卡因而帶有[[恐怖遊戲|恐怖遊戲]]元素<ref name="funoffear">{{cite journal |journal=[[PC_Gamer_UK|PC Gamer UK]] |title=All the Fun of the Fear |issue=88 |date=December 2000 |last=Gillen |first=Kieron |accessdate=2009-08-13 }}</ref><ref name="horror">{{cite web |archiveurl=https://web.archive.org/web/20041109234407/http://www.gamespot.com/gamespot/features/pc/history_horror_pt2/index.html |url=http://www.gamespot.com/gamespot/features/pc/history_horror_pt2/index.html |archivedate=2004-11-09 |title=Ghouls, Ghosts, and Long-Legged Beasts: A Modern History of Horror Games Part II |work=[[GameSpot|GameSpot]] |last=Todd |first=Brett |accessdate=2009-08-13 |deadurl=yes }}</ref>。

主角配備了一枝鈍頭棒和一口劍，分別能癱瘓或殺死非玩家角色，使用劍作正面交鋒時能作出三種直接攻擊和擋格<ref name="thiefmanual" />，倒下的非玩家角色可被抬走藏在其他地點<ref name="thiefmanual" />。主角也配備了一把能作為遠程武器或工具的弓，可發射能熄滅火把的「水箭」和放下繩索的「繩箭」等<ref name="thiefmanual" />。其他工具包括開鎖器、[[閃光彈|閃光彈]]、速度魔藥等<ref name="thiefmanual" />。工具可在關卡之間購買<ref name="cgsp">{{cite web |url=http://www.cdmag.com/articles/016/017/thief_review.html |title=''Thief: The Dark Project'': The quietly brilliant 3D action game |archiveurl=https://web.archive.org/web/20020827125444/http://www.cdmag.com/articles/016/017/thief_review.html |archivedate=2002-08-27 |work={{tsl|en|Computer Games Magazine}} |last=Cross |first=Jason |date=1998-12-10 |accessdate=2009-08-05 |deadurl=yes }}</ref>。

== 劇情 ==
===背景===
遊戲設定在包含[[中世紀|中世紀]][[黑暗奇幻|黑暗奇幻]]與[[工業革命|工業革命]]元素<ref name="gamepro" /><ref name="GAMESPY_HALL_OF_FAME">{{cite web |url=http://www.gamespy.com/articles/493/493004p1.html |title=Hall of Fame: Thief: The Dark Project |last=Carlson |first=Rich |publisher=[[GameSpy|GameSpy]] |date=2001-08-01 |accessdate=2009-08-19}}</ref>的大都會「城市」（The City）<ref name="gamepro" />。遊戲總監表示背景本質上是未介定的中世紀時代，像中世紀[[歐洲|歐洲]]、1985年電影《[[妙想天開|妙想天開]]》與1995年電影《{{link-en|童夢失魂夜|The City of Lost Children}}》的混合，遊戲內有[[電力|電力]]、少量[[魔法|魔法]]及部分19世紀機械技術<ref name="nextgenpreview">{{cite journal |journal={{tsl|en|Next Generation Magazine}} |title=The Dark Project |date=March 1997 |issue=27 |pages=54–55 }}</ref>，被認為是[[蒸汽引擎|蒸汽引擎]]技術被廣泛使用時期的[[蒸汽朋克|蒸汽朋克]]<ref name="arstechnica">{{cite web |url=http://arstechnica.com/gaming/news/2008/05/steampunk-video-games.ars |title=The memories of our future: Steampunk in gaming |last=Thompson |first=Michael |date=2008-05-28 |work=[[Ars_Technica|Ars Technica]] |accessdate=2009-07-30}}</ref>。背景設定也在關卡內的筆記和非玩家角色對話中有所交代<ref name="retrospective" /><ref name="escapist">{{cite web |url=http://www.escapistmagazine.com/articles/view/issues/issue_205/6150-Robbing-Gods |title=Robbing Gods |date=2009-06-09 |last=Hindmarch |first=Will |work=The Escapist |accessdate=2009-07-30}}</ref>。

城市的三大勢力為中立和維持平衡的「管理員」（Keepers）、追求秩序之「聖槌教團」（Hammerites）和追求混亂之「異端派」（Pagans）<ref name="retrospective" /><ref name="escapist" />。聖槌教團以「造物主」（Builder）為膜拜對象，崇尚文明進步、工藝和正義；異端派以「妖神」（Trickster）為膜拜對象，崇尚自然世界<ref name="retrospective" /><ref name="escapist" />。各派別之建築物被特別設計以反映其理念<ref name="escapist"/>。

=== 梗概 ===
主角加洛特（Garrett）為一名孤兒，兒時以偷竊及帶送口信為生，一天晚上向一名行蹤隱密之男子行竊{{efn|加洛特：「我當時是個孩子，無父無母無家可歸，以替人帶口訊和偷錢包維生。有一晚我看見一個男人，其他人只是從他身邊走過，像是他不存在似的。我覺得他一定有甚麼值錢的東西，所以我偷偷跟上去用手一抓。」}}事敗，男子自稱管理員成員並因認為加洛特有才能而邀其加入組織{{efn|阿提密斯：「那不是給你的。」加洛特：「求求你先生，我餓壞了。別告訴聖槌教團，我保證。」阿提密斯：「你叫甚麼名字？」加洛特：「加洛特。」阿提密斯：「你很有才能，小子。」加洛特：「放開我，老頭子！」阿提密斯：「要看見一個管理員並不容易，尤其是不希望被其他人發現的一個。我們需要像你這樣有才能的人。如果你已厭倦了你現時的生活方式就跟我來，我會向你展示一個新的方式。」加洛特：「別煩我了！」阿提密斯：「如你所願。」加洛特：「我在他即將消失於人叢中前追上了他，那是一段漫長教育的開始。」}}。加洛特接受各項匿蹤訓練後決定離開組織以偷竊為業{{efn|加洛特：「管理員希望把我訓練成他們一分子，但我覺得那些技巧有其他用途。」}}。

== 開發 ==
遊戲開發於1996年4月展開<ref name="cgm5">{{cite web |archiveurl=https://web.archive.org/web/20030908171031/http://www.cdmag.com/articles/025/191/dougc_interview.html |url=http://www.cdmag.com/articles/025/191/dougc_interview.html |title=The Tracks of His Games |date=2000-01-30 |last=Bauman |first=Steve |work={{tsl|en|Computer Games Magazine}} |archivedate=2003-09-08 |accessdate=2010-11-02 |deadurl=yes }}</ref>，原設計師兼編劇肯·利維恩（Ken Levine）被[[每日電訊報|每日電訊報]]譽為遊戲創作的靈魂人物<ref>Lewis Denby, [http://www.telegraph.co.uk/technology/video-games/8865074/BioShock-Infinite-interview-Ken-Levine.html BioShock Infinite interview - Ken Levine], Telegraph, 03 Nov 2011</ref>，設計靈感來自其最喜愛之兩款遊戲：《{{link-en|德軍堡壘|Castle Wolfenstein}}》與《[[暗黑破壞神|暗黑破壞神]]》<ref>Dan Ryckert, [http://www.gameinformer.com/b/features/archive/2012/05/30/things-you-didn-39-t-know-about-ken-levine.aspx?PostPageIndex=1 Things You Didn&#39;t Know About Ken Levine], www.GameInformer.com, May 30, 2012</ref>。開發概念本為製作一款[[角色扮演|角色扮演]][[動作遊戲|動作遊戲]]，因此利維恩被安排負責設計遊戲之世界與劇情。早期理念和開發項目成為《暗黑[[卡美洛|卡美洛]]》（Dark Camelot），期後才演變成為《暗黑計劃》。遊戲首個概念為一款第一人稱劍擊模擬遊戲：五十年代[[冷戰時期|冷戰時期]][[蘇聯|蘇聯]]被不死生物侵占，作為[[中央情報局|中央情報局]]僅存成員的玩家需要斬殺牠們<ref name=rpga>[http://rpgvaultarchive.ign.com/archive/19990207.shtml Vault Network News: Week of February 7, 1999] {{webarchive|url=https://web.archive.org/web/20110727090005/http://rpgvaultarchive.ign.com/archive/19990207.shtml |date=2011年7月27日 }}</ref>，而且可選擇是否與派別同盟或敵對<ref name=doug>Rouse R. Game, ''Design Theory and Practice'', 2005. [http://flylib.com/books/en/4.479.1.135/1/ Chapter 26: Interview: Doug Church]</ref>，但被市場部門否決<ref name=ken>[http://gambit.mit.edu/updates/2011/06/looking_glass_studios_intervie_4.php GAMBIT: Updates: Looking Glass Studios Interview Series - Audio Podcast 5 - Ken Levine]</ref>。

作為第二個概念之《暗黑卡美洛》仍以劍擊為核心，採用第一人稱視角，結合了動作遊戲、角色扮演遊戲和[[冒險遊戲|冒險遊戲]]元素<ref name="leonard1" />，劇情為[[亞瑟王傳說|亞瑟王傳說]]的相反：[[莫卓|莫卓]]是被曲解的英雄、[[亞瑟王|亞瑟王]]是暴君、[[梅林_(亞瑟王傳說)|梅林]]是精神病患者<ref name="makingof">The Making of ''Thief II'' featurette. ''Thief Gold''.</ref><ref name=dc>Craig Pearson, [http://www.rockpapershotgun.com/2013/05/17/pre-thief-dark-camelot-footage/ Pre-Thief: Dark Camelot Footage], Rock, Paper, Shotgun, May 17th, 2013.</ref>、[[摩根勒菲|摩根勒菲]]是莫卓之顧問、[[桂妮薇兒|桂妮薇兒]]是一名會背叛[[蘭斯洛特爵士|蘭斯洛特爵士]]協助莫卓潛進卡美洛偷取[[聖杯|聖杯]]之同性戀者<ref name=doug/>。遊戲美工丹·斯朗（Dan Thron）表示他們很長一段時間並不了解遊戲內容，直至有人突然提出玩家不應只是斬殺敵人的潛行概念<ref name=rpga>[http://rpgvaultarchive.ign.com/archive/19990207.shtml Vault Network News: Week of February 7, 1999] {{webarchive|url=https://web.archive.org/web/20110727090005/http://rpgvaultarchive.ign.com/archive/19990207.shtml |date=2011年7月27日 }}</ref>。遊戲設計師道格·卓治（Doug Church）表示當初的潛行構思為玩家趁著守衛不注意時盡快越過他們，設計師保羅·尼拉夫（Paul Neurath）認為潛行相當有趣並提出製作一款與盜賊有關的遊戲<ref name=doug/>。

《暗黑卡美洛》於1997年早期被更名為《暗黑計劃》，改以偷竊和潛行為核心，但部分已設計之關卡仍被採納使用<ref name="leonard1" />。遊戲總監於3月表示他們本質上在製作某類[[模擬遊戲|模擬遊戲]]，使用真實的物件互動和物理，首席遊戲設計謝夫·尤斯（Jeff Yaus）補充道他們的目標是使一切物件的表現合乎常理，例如會燃燒的物件將會燃燒，但是否這樣做交由玩家決定<ref name="nextgenpreview" />。潛行設計草案於4月4日完成<ref name=diary/>，其中「維持隱密時力量強大；現身時非常弱小」的理念來自[[潛艇|潛艇]]戰爭，特別是1985年遊戲《{{link-en|寂靜部隊|Silent Service}}》<ref name=ken/>。團隊曾計劃引進[[多人電子遊戲|多人模式]]，包括以[[死鬥|死鬥]]為基礎、各盜賊小隊於時限內搶先從非玩家角色領域中偷取財寶之「盜鬥」模式（Theftmatch）<ref>[http://www.thief-thecircle.com/darkproj/faq.html Frequently Asked Questions]</ref>。

《暗黑計劃》開發工作於1997年5月全面展開並趕及在同年[[E3|E3]]中展出遊戲示範和宣傳短片，原定夏季推出<ref name=diary>[http://www.thief-thecircle.com/darkproj/darkdiary.html Looking Glass Studios - Thief: The Dark Project (Project Diary)]</ref>，但工作室於1997年中面臨嚴重財政問題，其[[奧斯汀|奧斯汀]]分部關閉並導致數名人員離職，包括擔任《暗黑卡美洛》遊戲製作之華倫·史碧塔（Warren Spector）<ref name="euro2000">{{cite web |archiveurl=https://www.webcitation.org/5tnzHO3lu?url=http://www.eurogamer.net/articles/warrens1 |url=http://www.eurogamer.net/articles/warrens1 |title=Warren Spector of Ion Storm (Part One) |date=2000-03-31 |archivedate=2010-10-28 |work=[[Eurogamer|Eurogamer]] |accessdate=2009-08-17 |deadurl=yes }}</ref>和利維恩<ref name=ken/>。早於4月18日<ref name=diary/>，工作室已在半年內解聘了約半數顧員，開發團隊的士氣也因隊內成員大幅改變而受到打擊，首席遊戲程序師湯姆·里安納特（Tom Leonard）表示眾人承受巨大壓力，數名人員也因此自願離職，包括負責設計非玩家角色人工智能之前首席程序師布里斯高·羅謝斯（Briscoe Rogers）<ref name=diary/>，遊戲也因此出現蠕蟲和其他問題<ref name="leonard1" />。

里安納特在接任首席程序師一職後認為人工智能系統仍可修繕；他在數月間發現協助非玩家角色探索路徑之資料碼已無法挽救，並於1997年11月利用約五分之一的原碼完成了設計。為集中精力創造以潛行為中心之單人任務，里安納特於開發階段廢除了多人模式、複雜的道具介面和支線性任務結構，亦認為多人模式的廢除和遊戲從《暗黑計劃》更名為《神偷：暗黑計劃》能使團隊目標更明確<ref name="leonard1" />。遊戲名稱終在1998年4月3日變更，團隊亦提出和否決了數項構思，包括在牆壁和天花板上爬行的能力、縮小魔藥和隱身魔藥<ref name=diary>[http://www.thief-thecircle.com/darkproj/darkdiary.html Looking Glass Studios - Thief: The Dark Project (Project Diary)]</ref>。1998年夏季，團隊備受疲勞和遊戲內的模擬及人工智能漏洞困擾，里安納特之新人工智能系統亦暫停製作以給予團隊足夠時間把遊戲概念集結成試玩版，直至經過十二星期連續工作後才繼續<ref name="leonard1" />。

遊戲大部分問題在預計發行日期三個月前已被解決，里安納特表示團隊開始對遊戲抱有正面觀感，同時《{{link-en|魔鬼戰將：天擊戰線|Commandos: Behind Enemy Lines}}》、《[[戰慄時空|戰慄時空]]》和《[[潛龍諜影|潛龍諜影]]》等遊戲的推出減低了團隊對試驗性質遊戲銷量不佳的憂慮，長達兩年半和耗資三百萬元的開發工作終在熱情和自信下結束<ref name="leonard1" />。遊戲在1998年11月30日由[[Eidos_Interactive|Eidos Interactive]]發行，其加強版《神偷：黃金》（Thief Gold）由望遠鏡工作室與Eidos於1999年10月推出，增加三個新任務並修繕了原版本之十二個任務<ref name="allgame2">{{cite web |url=http://www.allgame.com/game.php?id=20170&tab=review |last=Nguyen |first=Cal |title=''Thief Gold'' Review |work=[[allgame|allgame]] |accessdate=2009-08-30}}</ref><ref name="cgsp2">{{cite web |archiveurl=https://web.archive.org/web/20020827125441/http://www.cdmag.com/articles/024/107/thiefg_review.html |url=http://www.cdmag.com/articles/024/107/thiefg_review.html |title=''Thief Gold'' |archivedate=2002-08-27 |date=1999-11-24 |last=Sones |first=Benjamin E. |work={{tsl|en|Computer Games Magazine}} |accessdate=2009-08-22 |deadurl=yes }}</ref>，遊戲光碟也包括[[關卡編輯器|關卡編輯器]]、續作《神偷Ⅱ：金屬時代》製作花絮影片等額外內容<ref name="allgame2" />。

=== 設計 ===
遊戲設計以第一人稱潛行和避免交鋒作為核心，里安納特表示這構思是對傳統第一人稱射擊遊戲的衝擊，不少人擔心這遊戲風格將不能吸引玩家，甚至深入參與遊戲開發的人員亦曾有此疑慮<ref name="leonard1" />。對於玩家表示需要投放一段時間才能享受團隊以前開發的遊戲，團隊修改《暗黑計劃》的設計讓玩家能在剛接觸遊戲就已經掌握遊玩方式<ref name="dougchurch" />；卓治表示團隊的目標是把遊戲設計發揮至極限<ref name="leonard1" /><ref name="GAMESPY_HALL_OF_FAME" />但同時與工作室以往遊戲的核心設計相同，希望提供一個玩家能利用一系列能力選擇自己的目標的世界，讓他們按各自的方式克服障礙和使他們在目標完成後更感滿足，而彈性的遊戲元素模擬是有效的方法<ref name="dougchurch">{{cite web |url=http://www.cdmag.com/articles/025/183/dougc_interview.html |archiveurl=https://web.archive.org/web/20020827130124/http://www.cdmag.com/articles/025/183/dougc_interview.html |archivedate=2002-08-27 |title=Doug Church |work={{tsl|en|Computer Games Magazine}} |date=2000-01-28 |last=Bauman |first=Steve |accessdate=2009-08-02 |deadurl=yes }}</ref>。

遊戲主要被設計成非預設；任務內各事件大多自然發生。里安納特表示此舉乃希望加強玩家互動與即興<ref name="leonard1" />，玩家與非玩家角色的關係正是遊戲內的主要障礙<ref name="leonard1" />，為此遊戲需要先進的人工智能系統。里安納特指出如《戰慄時空》等身一人稱射擊遊戲的人工智能大多運用了「觀察與聆聽」系統（look and listen systems），非玩家角色在看見或聽見玩家後會變得凶暴，《暗黑計劃》之系統擴大了非玩家角色能感受的狀態，例如它們在聽見可疑的聲音變成完全敵對前能先作出調查<ref name="leonard2" />。

遊戲設計蘭迪·史密斯（Randy Smith）表示光與影是遊戲中的安全界線，但這界線並不穩定或完全安全，玩家最終需要離開安全區域，並在進入另一安全區域前面對危機，即使躲藏起來也會感到不安，但隨著技術進步將能判斷安全的程度<ref name="funoffear" />。部分任務包含恐怖遊戲元素<ref name="horror" />，其中任務九「回到教堂」（Return to the Cathedral）故意使玩家無法判斷他們是否安全<ref name="funoffear" />，史密斯認為迫使玩家非自願地面對恐懼更能加強內心的恐懼，該任務也迫使玩家在整個任務中作出一切有可能作出的恐怖之事。雜誌《PC Gamer UK》表示這任務創造了一個鬆懈和緊張的循環，強烈地捶打著玩家的[[神經末梢|神經末梢]]<ref name="funoffear">{{cite journal |journal=[[PC_Gamer_UK|PC Gamer UK]] |title=All the Fun of the Fear |issue=88 |date=December 2000 |last=Gillen |first=Kieron |accessdate=2009-08-13 }}</ref>。

任務以切合劇情而設計而非改變劇情以迎合任務<ref name="makingof">The Making of ''Thief II'' featurette. ''Thief Gold''.</ref>，團隊從1997年遊戲《[[黃金眼007|黃金眼007]]》獲得啟發而加入了一個能改變任務目標的難度系統，使設計師能在各遊戲難度設計不同的遊戲體驗而不需要改變各任務的整體結構，從而以最低開發成本提高重玩價值<ref name="leonard1" />。團隊基於這概念而提高玩家於較高難度設定中的殺敵限制；編劇兼聲優泰莉·布羅斯易斯（Terri Brosius）表示他們花了不少努力確保所有任務均可在不殺一人的情況下完成<ref name="salon2">{{cite web |url=http://dir.salon.com/tech/feature/2000/06/20/dark_glass/ |title=Game over |last=Au |first=Wagner James |date=2000-06-20 |publisher={{tsl|en|Salon.com}} |accessdate=2009-08-02 }}</ref>。

遊戲總監葛里格·羅彼高羅（Greg LoPiccolo）希望遊戲聲效能使環境和遊玩過程更豐富，因此遊戲需要一個先進的聲效系統。設計師為每個任務創造了一個「空間資料庫」（room database）以營造真實的聲波傳播<ref name="leonard1" />；遊戲音樂作家艾力克·布羅斯易斯（Eric Brosius）與團隊希望聲效能作為非玩家角色位置與狀態的提示，同時作為非玩家角色對玩家位置的提示，為此他們為非玩家角色錄製了語音，相反物件發出的聲音能使非玩家角色掌握主角的位置，非玩家角色之間也能透過聲效溝通：守衛的呼叫能警告一定範圍內的其他守衛<ref name="leonard1" />。聲音也作為交代劇情的元素：著重潛行的玩家能竊聽非玩家角色之間的對話以對遊戲背景有更深入的了解
<ref name="gamesnothollywood" />。

=== 引擎 ===
遊戲採用之{{link-en|黑暗引擎|Dark Engine}}並非獨立預算項目而是於開發階段內編寫<ref name="leonard1" />，偏重模擬現實物理，例如[[箭|箭]]發射後會以[[拋物線|拋物線]]下墜而非直線飛行<ref name="IGN_THIEF_PREVIEW">{{cite web |url=http://pc.ign.com/articles/123/123476p1.html |title=Thief: The Dark Project preview |author=IGN staff |publisher=IGN |date=1998-10-07 |accessdate=2009-07-29 }}</ref>，亦有{{link-en|alpha混合|alpha blending}}、[[紋理濾波|紋理濾波]]、動態光源和[[動作捕捉|動作捕捉]]<ref name="MAXIMUM_PC_REVIEW">{{cite journal |title=Thief: The Dark Project review |last=Sanchez |first=Andrew |journal={{tsl|en|Maximum PC}} |issue=7 |date=February 1999 |page=79 }}</ref>。[[渲染|渲染]]大部分由望遠鏡工作室程序師西恩·巴烈特（Sean Barrett）於1995年秋季編寫並預計在遊戲發售前完成。巴烈特在1996年離職後以合約形式繼續編寫，亦協助硬件支援等工作，但一直未有全面地處理渲染，其完成度也比同期其他渲染低<ref name="leonard1" />。

黑暗引擎被設計成可以重用和方便程序師給合完成品，勒布朗克根據這概念而編寫了「暗黑物件系統」（Dark Object System）。里安納特表示這系統是管理遊戲模擬內各種獨立物品的資料庫<ref name="leonard1" />，設計師能透過操縱物品改變遊戲行為而不需編寫額外編碼，系統也管理源頭數據以及材質、關卡、[[3D模型|3D模型]]、聲效等遊戲內容<ref name="leonard1" />。一個未完成之黑暗引擎被用作與遊戲開發者[[Irrational_Games|Irrational Games]]合作開發出1999年遊戲《{{link-en|網路奇兵2|System Shock 2}}》，由於暗黑物件系統表現良好，《暗黑計劃》與《網路奇兵2》在大部分開發過程中使用了相同的[[執行檔|執行檔]]<ref name="leonard1" />。

遊戲直至2000年5月售出超過五十萬份，成為望遠鏡工作室商業上最成功的遊戲<ref>{{cite news |url=http://www.highbeam.com/doc/1P2-8603067.html |title=Looking Glass, Closing, Laying of 60 Workers / Failed Deal Dooms Cambridge Game Developer |last=Bray |first=Hiawatha |date=2000-05-26 |work=[[The_Boston_Globe|The Boston Globe]] |page=C3 |accessdate=2009-08-03 |archive-url=https://web.archive.org/web/20150329163947/http://www.highbeam.com/doc/1P2-8603067.html |archive-date=2015-03-29 |dead-url=yes }}</ref>。

遊戲的超自然與洞穴探索元素引來批評<ref name="cgsp" /><ref name="presley" /><ref name="Au" />，部分評論認為應採用較真實和以偷竊為核心的任務。《PC Zone》認為不死生物使遊戲淪為市面上眾多[[砍殺遊戲|砍殺遊戲]]之其中一作，破壞了劇情敍述<ref name="presley">{{cite web |last=Presley |first=Paul |title=PC Review: ''Thief: The Dark Project'' |publisher=[[Computer_and_Video_Games|Computer and Video Games]] |work={{tsl|en|PC Zone}} |archiveurl=https://web.archive.org/web/20071016064158/http://computerandvideogames.com/article.php?id=3549&skip=yes |url=http://www.computerandvideogames.com/article.php?id=3549&skip=yes |archivedate=2007-10-16 |date=2001-07-13 |accessdate=2006-11-18 |deadurl=yes }}</ref>；《PC Gamer UK》認為部分關卡與《[[盜墓者|盜墓者]]》相近但算不上出色 <ref name="pcgameruk" />；《電腦遊戲世界》認為部分關卡難度過高<ref name="cgw" />；雜誌《{{link-en|Next Generation|Next Generation}}》表示雖然潛行或會過於重複，但遊戲仍然有趣而且對其類別而言是有價值的一作<ref name="nextgen" />。

=== 影響 ===
《神偷：暗黑計劃》是第一款[[3D|3D]]潛行[[個人電腦|個人電腦]]遊戲，其潛行設定對後期同類別遊戲造成影響<ref name="GAMERADAR_SNEAKY_HISTORY">{{cite web |url=http://www.gamesradar.com/f/the-sneaky-history-of-stealth-games/a-2009020393535662028 |title=The sneaky history of stealth games |last=Patterson |first=Shane |publisher=[[GamesRadar|GamesRadar]] |date=2009-02-03 |accessdate=2009-08-19}}</ref>，也被認為是第一款以光與影作潛行機制<ref name="GAMESPY_HALL_OF_FAME">{{cite web |url=http://www.gamespy.com/articles/493/493004p1.html |title=Hall of Fame: Thief: The Dark Project |last=Carlson |first=Rich |publisher=[[GameSpy|GameSpy]] |date=2001-08-01 |accessdate=2009-08-19}}</ref>及以聲音作環境提示的遊戲<ref name="GAMERADAR_SNEAKY_HISTORY" />，其中容許聲音穿越角落和房間的聲波傳播系統引起遊戲開發商的廣大回響，後期如《[[刺客教條|刺客教條]]》、《[[刺客任務|刺客任務]]》、《[[縱橫諜海|縱橫諜海]]》和《[[天誅|天誅]]》等潛行遊戲均有《暗黑計劃》之影子<ref name = "GAMEPRO_IMPORTANT">{{cite web | url = http://www.games.net/article/netten/1/112821/the-10-most-important-modern-shooters/ | modern shooters | author = Shuman, Sid | publisher = [[GamePro|GamePro]] | accessdate = 2009-08-19}}</ref>。《戰慄時空》遊戲設計兼編劇馬克·尼羅（Marc Laidlaw）形容《暗黑計劃》是他所玩過最恐怖、最引人入勝和最有滿足感的遊戲，亦是他一直重玩和最喜歡的單人遊戲<ref name="GAMINGNEXUS_LAIDLAW_INTERVIEW">{{cite web |url=http://www.gamingnexus.com/Default.aspx?Section=Article&I=801 |title=Valve Software interview: Marc Laidlaw |last=Yan |first=John |publisher={{tsl|en|GamingNexus.com}} |date=2003-10-27 |accessdate=2009-08-19 |deadurl=yes |archiveurl=https://web.archive.org/web/20070930185312/http://www.gamingnexus.com/Default.aspx?Section=Article&I=801 |archivedate=2007-09-30 }}</ref>，並表示《暗黑計劃》是他希望曾參與製作的遊戲<ref name="GAMESPY_HALL_OF_FAME" />；《[[輻射3|輻射3]]》首席設計師亦表示《暗黑計劃》是他最喜歡的遊戲<ref name="GAMESPOT_EMIL">{{cite web |url=http://www.gamespot.com/pc/action/thiefthedarkproject/news.html?sid=6206965&om_act=convert&om_clk=gsupdates&tag=updates;title;4 |title=GDC 2009: Fallout 3 lead opens game design vault |author=GameSpot staff |publisher=[[GameSpot|GameSpot]] |date=2009-03-27 |accessdate=2009-08-19}}</ref>。

《暗黑計劃》被部分刊物列為史上最偉大遊戲之一。《[[GameSpy|GameSpy]]》表示遊戲有別於當時大部分[[第一人稱射擊遊戲|第一人稱射擊遊戲]]，鼓勵潛行和避免正面交鋒，否定了動作遊戲必須是射擊類別的想法，為這已經數量過剩的遊戲類別開闢了全新的世界<ref name="GAMESPY_HALL_OF_FAME" />；《GameSpot》表示《潛龍諜影》、《天誅》和《暗黑計劃》為潛行動作遊戲下了定義，但只有後者對利用影子前進的玩法作出了最完美的闡述，亦是電子遊戲界中仍未超越的成就<ref name="GAMESPOT_GREATEST_GAME">{{cite web |url=http://www.gamespot.com/gamespot/features/all/greatestgames/p-33.html |title=The Greatest Games of All Time: ''Thief: The Dark Project'' |last=Kasavin |first=Greg |work=[[GameSpot|GameSpot]] |accessdate=2007-10-22}}</ref>；[[IGN|IGN]]於2009年把《暗黑計劃》列進其名遊戲榜<ref>{{cite web |url=http://games.ign.com/halloffame/thief-the-dark-project.html |title=Videogame Hall Of Fame: Thief: The Dark Project |author=IGN staff |work=[[IGN|IGN]] |accessdate=2009-10-19}}</ref>；《GamePro》表示《暗黑計劃》是潛行動作遊戲的先鋒<ref name = "GAMEPRO_IMPORTANT"/>；《[[Eurogamer|Eurogamer]]》表示近代潛行遊戲只是笨拙地模仿《暗黑計劃》這大師級原創遊戲，並應為此感到羞恥<ref>John Walker, [http://www.eurogamer.net/articles/retrospective-thief-the-dark-project-article Retrospective: Thief The Dark Project Article], Eurogamer.net, 17 May 2009</ref>；《[[Kotaku|Kotaku]]》於2012年表示《暗黑計劃》是最佳的潛行遊戲，比任何近代同類別遊戲優秀<ref>Mike Fahey, [http://kotaku.com/5897240/best-game-ever-thief-the-dark-project-is-the-only-way-to-stealth Best Game Ever: Thief: The Dark Project is the Only Way to Stealth], Kotaku, 28 March 2012</ref>；[[時代雜誌|時代雜誌]]於同年把《暗黑計劃》列為史上百大最偉大遊戲之一<ref name="AccoTimeAllTime">{{cite news| first=| last=| title=All-TIME 100 Video Games| url=http://techland.time.com/2012/11/15/all-time-100-video-games/slide/all/| work=Time| publisher=Time Inc.| date=2012-11-15| accessdate=2012-11-15| archiveurl=https://www.webcitation.org/6CCgwlecQ?url=http://techland.time.com/2012/11/15/all-time-100-video-games/slide/all/| archivedate=2012-11-15| deadurl=no}}</ref>。

遊戲有兩款續作以及一款重啟作。望遠鏡工作室於2000年3月推出獲得正面評價之《神偷Ⅱ：金屬時代》，由[[離子風暴|離子風暴]]奧斯汀工作室開發之《神偷：致命暗影》於2004年5月推出，同樣獲得正面評價；2009年5月，消息透露[[Eidos蒙特利爾|Eidos蒙特利爾]]開發[[神偷_(遊戲)|第四款神偷系列作品]]，對應微軟視窗、[[PlayStation_3|PlayStation 3]]和[[Xbox_360|Xbox 360]]<ref name="IGN_THIEF4_CONFIRMED">{{cite web |url=http://xbox360.ign.com/articles/981/981496p1.html |title=''Thief 4'' confirmed |first=Charles |last=Onyett |publisher=IGN |date=2009-05-11 |accessdate=2009-08-19}}</ref>，評價為中至正面。《暗黑計劃》自望遠鏡工作室關閉後由玩家團體進行支援，當中包括以近代遊戲引擎重現《暗黑計劃》神髓之[[遊戲模組|遊戲模組]]《{{link-en|暗黑模組|The Dark Mod}}》<ref>{{cite web|title=The Dark Mod - Stealth Gaming in a Gothic Steampunk World|url=http://www.thedarkmod.com/main/|work=thedarkmod.com|accessdate=2013-12-26}}</ref>，原定作為《[[毀滅戰士3|毀滅戰士3]]》之遊戲模組於2009年推出，最後在2013年10月作為[[開放原始碼|開放原始碼]]獨立遊戲推出<ref>{{cite web|last=Pearson|first=Craig|date=2013-10-09|title=Praise The Builder: The Dark Mod Is Standalone|url=http://www.rockpapershotgun.com/2013/10/09/praise-the-builder-the-dark-mod-is-standalone/|work=[[Rock,_Paper,_Shotgun|Rock, Paper, Shotgun]]|accessdate=2013-12-26}}</ref>；2013年12月，玩家團體推出[[高清|高清]]模組《神偷：黃金高清》（Thief Gold HD）<ref>{{cite web|last=Smith|first=Adam|date=2013-12-11|title=Sharper Shadows: Thief Gold HD Mod Released|url=http://www.rockpapershotgun.com/2013/12/11/sharper-shadows-thief-gold-hd-mod-released/|work=[[Rock,_Paper,_Shotgun|Rock, Paper, Shotgun]]|accessdate=2013-12-26}}</ref>。
