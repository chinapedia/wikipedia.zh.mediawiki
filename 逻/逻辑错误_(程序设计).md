在计算机[[程序设计|程序设计]]中，'''逻辑错误'''('''Logic error''')（又叫-{zh-hans:'''语义错误''';zh-hant:'''語意錯誤'''}-）是[[程序错误|程序错误]]的一种，这种错误会导致程序运行出错，但还不至于崩溃。逻辑错误会产生意外的输出或结果，但并不一定会立即被发现。

逻辑错误在[[编译器|编译语言]]和[[解释器|解释语言]]中都可能出现。与[[语法错误|语法错误]]的程序不同的是，逻辑错误的程序从语法上来说是正确的一段程序，但其运行结果却与预期不符。<ref>{{cite web|url=http://msdn.microsoft.com/zh-cn/library/s9ek7a19(v=vs.90).aspx|title=了解您的 Bug：三种编程错误|publisher=Microsoft}}</ref>逻辑错误的唯一表现就是错误的运行结果。

在 C++ 语言中，逻辑错误是一种特殊的[[异常处理|异常]]（''logic_error''）。

== 常见原因 ==
逻辑错误的原因可能是[[陳述_(程式)|语句]]中出现了差错（比如公式写错）、[[算法|算法]]中的错误，甚至是选择了错误的算法。此外还有一些其它原因，如类型转换错误、变量作用域错误、遗漏的代码段、对问题或需求的理解有误等等。

== 调试逻辑错误 ==
找出逻辑错误的办法之一，是将程序中的[[变量_(程序设计)|变量]]输出到文件或打印到屏幕，以定位错误所在。虽然这个办法不是所有情况下都适用，比如在调用了错误的[[子例程|子例程]]时就行不通，但是如果程序由于[[算法|计算结果]]有误而出错，这种方法最容易找出问题。

== 示范 ==
以下用于计算[[平均数|平均数]]的[[C语言|C语言]]函数含有逻辑错误。代码中缺少了一对括号，因此程序虽然能够通过编译并运行，但由于[[运算符优先级|运算符优先级]]（先乘除，后加减）的缘故，运算结果并不正确。
<source lang="c">
int average(int a, int b)
{
    return a + b / 2;     /* 应为 (a + b) / 2 */
}
</source>
