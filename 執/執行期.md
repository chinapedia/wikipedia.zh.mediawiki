'''執行時期'''（{{lang|en|Run time}}）在[[電腦科學|電腦科學]]中代表一個[[電腦程式|電腦程式]]從開始執行到終止執行的運作、執行的時期。與執行時期相對的其他時期包括：[[設计時期|設计時期]]（design time）、[[編譯時期|編譯時期]]（compile time）、[[链接期|鏈結時期]]（link time）、與[[載入時期|載入時期]]（load time）。

而[[運行環境|執行環境]]是一種為正在執行的程序或程式提供軟體服務的[[虚拟机|虛擬機械環境]]。它有可能是由作業系統自行提供，或由執行此程式的母程式提供。

通常由[[作業系統|作業系統]]負責處理程式的載入：利用[[載入器|載入器]]（loader）讀入程式碼，接著進行基本的[[記憶體|記憶體]]配置，並視需要[[链接器|聯結]]此程式指定的所有動態链接庫。有些程式語言會由此語言提供的運行環境處理上述工作。

<!--雖然早期在主流語言與常見的作業系統上不常見到執行期環境這樣的作法，主要只有 Java 語言使用執行期環境，但當時並不普及。由於微軟近年大力推動 .net framework、Windows 下 Visual Studio 開發環境的獨大、產出碼可較傳統方式精簡、執行期環境的效率較以往提升等因素，執行期環境已相當普及，不能算不常見。-->
程式碼的某些問題，只能在執行期間進行偵錯動作（或較有效率），例如[[演算法|邏輯錯誤]]或[[陣列|陣列]]邊際檢查等便屬此類。因此不管撰寫與測試得多麼精細，有些[[程序错误|錯誤]]必須在實際上線並處理真實資料的情況下才能找出。因此，程式使用者也許會遇到諸如'''執行時期錯誤'''之類的訊息。
