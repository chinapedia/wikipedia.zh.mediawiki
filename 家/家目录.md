'''家目录'''是在[[多用户|多用户]][[操作系统|操作系统]]上包含该系统的特定用户的[[计算机文件|文件]]的[[目录_(文件系统)|文件系统目录]]。家目录的具体内容（如它的名称和位置）是由操作系统参与定义的，由[[环境变量|环境变量]]存储；例如： [[Microsoft_Windows|Windows]]系统在[[Windows_2000|2000]]与[[Windows_2003|2003]]之间的版本中将家目录放在称为 ''Documents and Settings'' 的目录中、Windows Vista 以上系统家目录在 Users 文件夹中、Linux系统在 /home 中。

== 描述 ==
用户的家目录用于存放用户的文件；包括文本文档、音乐、图像或视频等，也可能包括一些记录用户使用偏好、软件首选设置的[[配置文件|配置文件]]：如[[浏览器|浏览器]][[书签_(浏览器)|书签]]，喜爱的桌面[[電腦壁紙|壁纸]]和[[主题|主题]]、特定软件为访问外部服务而保存的[[密码|密码]]等。用户可以在该目录中安装可执行软件，但这些软件只能向拥有该目录权限的用户开放。 家目录可以进一步组织子目录来使用。

用户家目录的内容由[[文件系统权限|文件系统权限]]保护，一般默认只有用户自身和管理人员有访问的权限。 任何获得管理权限的用户都有权访问[[文件系统|文件系统]]上任意受保护的内容，包括其他用户的家目录。

== 家目录的优点 ==
家目录可避免从整个系统中分离出冗余的用户数据，且让重要文件的备份操作相对简单。此外，在大多数情况下，以受感染用户的名义，并在他们的特权下运行的[[特洛伊木马_(电脑)|木马]]、[[计算机病毒|病毒]]和[[蠕虫病毒|蠕虫]]，只能够对受感染用户的家目录下的文件进行修改，或许其中一些文件属于用户所参与的工作组，但那些文件并不是真正的[[系统文件|系统文件]]。

== 各操作系统的其他特性 ==

===Unix===
在Unix下，用户在登录后自动被置于他们的家目录中。缩写变量<tt>~user</tt>指向用户的家目录(允许用户从文件系统中的任意位置浏览它或者在其他的Unix命令中使用。<tt>~</tt> ([[波浪号|波浪号]]字符)指向当前用户的家目录。

Unix的超级用户有权访问文件系统的任意目录，从而可以访问所有用户的家目录。在旧系统中超级用户的家目录为 <tt>[[根目录|/]]</tt> 但是在许多新的系统中，它位于<tt>[[家目录/root|家目录//root]]</tt> (Linux, BSD)或是<tt>[[家目录/var|家目录//var]]/root</tt> ([[Mac_OS_X|Mac OS X]]).

===VMS===
在[[VMS|VMS]]操作系统中，用户的家目录被称为“根目录”，同样地Unix/DOS/Windows/AmigaOS 的“根目录”被称为“主文件目录”。

== 对比单用户操作系统 ==
单用户操作系统只需为所有用户文件建立一个单一目录或者[[磁盘分区|分区]]，并没有针对单个用户而设置的独立目录（虽然用户仍然可以在主工作目录下手动设置和维护目录）。

* [[AmigaOS|AmigaOS]] 版本 2 及更高版本在硬盘上默认有"System"和"Work"分区。 
* [[BeOS|BeOS]] (及其继承系统) 有 /home 目录，其中包含系统上单用户的文件。 
* 在[[Windows_95|Windows 95]] OEM Service Release 2 之前的Windows版本中没有用户目录，但自此版本发布以来，<tt>\[[My_Documents|My Documents]]</tt>成为了单用户的家目录。
* [[NeXTSTEP|NeXTSTEP]]和[[OPENSTEP|OPENSTEP]]在单用户无网络安装中，<tt>/me</tt> 被使用，当作为超级用户登录时，<tt>/root</tt>被作为家目录。
