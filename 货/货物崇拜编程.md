'''货物崇拜编程'''（Cargo Cult Programming）是一种[[计算机程序设计|计算机程序设计]]中的[[反模式|反模式]]，其特征为不明就里地、仪式性地使用代码或程序架构。货物崇拜编程通常是[[程序员|程序员]]既没理解他要解决的[[bug|bug]]、也没理解表面上的解决方案的典型表现。{{r|Raymond}}

这个名词常指不熟练的或没经验的程序员从某处拷贝代码到另一处，却並未了解其代码是如何工作的，或者不清楚在新的地方是否需要这段代码。也可以指不正确或过分地应用[[设计模式_(计算机)|设计模式]]、代码风格或编程方法，却不清楚其原理。货物崇拜编程也可以指对[[设计模式_(计算机)|设计模式]]的盲目崇拜：盲目地使用设计模式，却不理解特定设计模式背后的原则。

== 语源 ==
[[货物崇拜|货物崇拜]]一词指二战期间太平洋某些原住民社会中产生的信仰。他们会建造模仿飞机形状的模型和形似飞机跑道的设施，期待能够召唤飞机，像战争时代一样给他们带来物资。{{r|Bezroukov|McConnell}}

在软件开发的语境中使用货物崇拜一词，可能源自物理学家[[理查德·费曼|理查德·费曼]]所提出的“[[草包族科学|货物崇拜科学]]”。{{r|McConnell|Jargon}}

== 货物崇拜软件工程 ==
“货物崇拜软件工程”（Cargo Cult Software Engineering）是一个与货物崇拜编程相关的概念。{{r|McConnell2003|page=23-26}}这个概念指盲目模仿成功的软件开发团体的表面现象，例如机械套用[[软件开发过程|软件开发过程]]却不理解其中的理由；或者因为一些成功的软件项目的开发团体出于热忱和激情而有较多无报酬的加班，而将无报酬加班当作成功的原因。

提出这一概念的史蒂夫·麦克康奈尔（Steve McConnell）认为，决定项目成功与否的根本条件是能力，而不是表面上的方法；他进一步指出，机械模仿成功者表象的“东施效颦的组织”是货物崇拜软件工程的成因。{{r|McConnell}}
